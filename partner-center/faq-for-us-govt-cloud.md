# <a name="frequently-asked-questions-for-partner-center-for-microsoft-cloud-for-us-government"></a>有关 Microsoft Cloud for US Government 合作伙伴中心的常见问题解答

**适用于**

-  Microsoft Cloud for US Government 合作伙伴中心

## <a name="frequently-asked-questions"></a>常见问题解答 

使用这些常见问题解答了解关于 Microsoft Cloud for US Government 云解决方案提供商 (CSP) 计划的详细信息。 你还可以联系你的帐户管理员以了解更多信息。 

### <a name="common"></a>通用

**Microsoft Cloud for US Government 是什么？**

针对美国政府而全新设计的政府 Microsoft 云允许美国境内的公共部门客户（大到联邦机构，小到镇政府）从一系列云计算服务中进行选择。 为了更好地满足客户的特定需求，Microsoft 已在数据中心中进行了重大投资，并专门满足美国联邦和州政策、法规和要求。 

**谁有资格参与 Microsoft Cloud for US Government 云解决方案提供商计划？**

位于美国境内，为美国联邦、州和地方政府实体提供政务能力服务的合作伙伴有资格参加 Microsoft 政府云的云解决方案提供商计划。 Microsoft 实施严格的合作伙伴资格验证程序，合作伙伴须通过验证方可访问 Microsoft 政府云。 若要取得资格，合作伙伴需要提供其与政府客户之间的合作关系证明。 美国政府 Microsoft 云 CSP 计划的注册流程已经过强化，以捕获更多数据来执行验证。 证明类型可以包括但不限于政府合同号和政府客户出具的担保函。

**客户定价是否与商业云解决方案提供商计划相同？**

否。 提供 Microsoft Cloud for US Government 的独特价值所需的额外成本会反映到定价中。

**合作伙伴折扣模式是否相同？**

是的。 Microsoft Cloud for US Government 云解决方案提供商计划的合作伙伴折扣模式与商业云解决方案提供商计划相同。

**CSP 商业版是一个现有的合作伙伴计划。Microsoft 政府云的 CSP 计划有何不同？**

Microsoft 政府云 CSP 计划与商业 CSP 之间存在以下不同点：

-   资格：Microsoft 政府云的云解决方案提供商计划仅面向位于美国境内，为美国联邦、州和地方政府实体提供政务能力服务的合作伙伴。 Microsoft 实施一套合作伙伴资格验证程序，合作伙伴须通过验证方可访问美国政府云。

-   加入：合作伙伴需要从商业云解决方案提供商计划单独加入 Microsoft Cloud for US Government 云解决方案提供商计划。

-   产品可用性：Azure 政府和基于席位的服务（如 Office 365、企业移动性 + 安全性和 Dynamics 365）均可在 Microsoft 政府云的云解决方案提供商计划中使用。 

-   合同：Microsoft 云经销商协议 (MCRA) 已经更新，以体现独特的政府条款。

-   合作伙伴中心功能：为了满足监管和合规要求，将不会为美国政府 Microsoft 云 CSP 计划启用合作伙伴中心的某些功能。 合作伙伴可以改用 Azure Active Directory (Azure AD) 或 [Graph API](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS) 来执行与用户和组管理相关的任务。 在美国政府 Microsoft 云 CSP 提供基于席位的服务后，才能启用支持多渠道和多合作伙伴方案的“关系请求”功能。

**我还不是 CSP 合作伙伴。在哪里可以找到有关如何成为 CSP 合作伙伴的详细信息？**

若要了解有关面向合作伙伴的注册模式的详细信息，请参阅 [Microsoft 合作伙伴网络中的云解决方案提供商](https://partner.microsoft.com/cloud-solution-provider)。

**许可工作原理**

*Microsoft 云经销商协议 (MCRA)*

MCRA 授权合作伙伴通过 CSP 进行转售。 加入美国政府 Microsoft 云即表示接受此协议的美国政府云版本。 此版本在标准云经销商协议后增添了美国政府云附录和附件 A。

美国政府 Microsoft 云的 MCRA 与标准 MCRA 之间存在以下两个不同点：

-   美国政府云附录包含适用于美国政府云服务交易的合作伙伴额外责任。

-   展示包含 Microsoft 客户协议（Microsoft 客户协议），必须将这些协议纳入到每个合同中，以便在 CSP 和客户之间输入到每个合同中的政府云服务。

*Microsoft 客户协议（Microsoft 客户协议）在 MCRA 中显示为展示形式*

CSP 需要将附件 A 并入其与购买美国政府 Microsoft 云的客户之间签订的政府合同。 此协议在合作伙伴中心发布，提供英文版。

适用于美国政府云的 Microsoft 客户协议不同于下面两种标准 Microsoft 客户协议：

-   需要将其添加到 CSP 的客户中。

-   前者已经过修改，符合联邦、州和地方法律法规的规定。

*Microsoft 云分销商协议 (MCDA)*

Microsoft 云分销商协议 (MCDA) 授权分销商通过云解决方案提供商计划进行销售。 MCDA 的美国政府云版本在标准 Microsoft 云经销商协议 (MCRA) 后面增添了美国政府云附录和附件 A。

美国政府 Microsoft 云的 MCDA 与标准 MCDA 之间存在以下两个不同点：

-   美国政府 Microsoft 云附录包含适用于美国政府云服务交易的合作伙伴额外责任。

-   展示 A 包含 Microsoft 客户协议（Microsoft 客户协议），需要将其加入到在 CSP 及其客户之间输入到的美国政府云服务的每个合同中。

**销售美国政府 Microsoft 云需要具备哪些资格？**

销售 Microsoft 政府云 CSP 计划所需的资格标准与商业 CSP 的资格标准相同，且同时对直接合作伙伴和间接经销商开放。 如果你已被批准成为商业 CSP 的间接经销商，则不需要再指定为 Microsoft 政府云 CSP 计划的间接经销商。

**我是现有合作伙伴。为美国政府销售 Microsoft 云需要执行哪些操作？**

合作伙伴需要加入美国政府 Microsoft 云 CSP 计划，以达到监管和合规要求。 （Microsoft Cloud for US Government 的加入与商业云解决方案提供商计划的加入相互独立。）加入后，你可以在 Microsoft Cloud for US Government 内创建客户并交易产品或服务。 如果要使用 API，请参阅 [Microsoft 国家云合作伙伴中心开发](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS)。

**合作伙伴奖励机制是否有任何不同？**

否。 Microsoft Cloud for US Government 云解决方案提供商计划的合作伙伴奖励机制与商业云解决方案提供商计划的合作伙伴奖励机制相同。 若要了解有关合作伙伴奖励机制的详细信息，请登录 [Microsoft 合作伙伴网络门户](https://partner.microsoft.com/membership/partner-incentives)。

**计费方式是否有任何变化？**

否。 Microsoft Cloud for US Government 云解决方案提供商计划的计费标准和要求与商业云解决方案提供商计划的计费标准和要求相同。 若要了解有关计费方式的详细信息，请查看 [Microsoft 合作伙伴网络](https://partner.microsoft.com/cloud-solution-provider/resources)上的资源。

**在商业数据中心托管的数据是否可以迁移到 Microsoft Cloud for US Government？**

为了促进并维护美国政府 Microsoft 云隔离的完整性，没有任何机制可以将客户的现有服务迁移到美国政府的 Microsoft 云中。 如果客户或合作伙伴使用 Azure，则应对迁移至美国政府 Microsoft 云内的新环境中的任何数据负责。


### <a name="microsoft-cloud-for-us-government-for-azure"></a>适用于 Azure 的 Microsoft Cloud for US Government

**适用于 Azure 的 Microsoft Cloud for US Government 是什么？**

Azure 政府云提供基于安全、隐私和控制、合规性和透明度等基本原则构建的云平台。 公众部门实体可接收 Microsoft Azure 的物理隔离实例，在该实例体系结构上构建的所有系统和应用程序均采用世界一流的安全和合规服务，这对于美国政府而言至关重要。 这些服务包括 FedRAMP 和 DoD 合规性认证、CJIS 国家级协议、颁发 HIPAA 业务合作协议的能力，且支持 IRS 1075。 Azure 政府由经筛选的美籍人员运营，支持多种用于在本地或云中构建和部署解决方案的混合方案。 公共部门实体还可利用超大规模云服务的即时可伸缩性和保证可用性。

**Microsoft Cloud for US Government 云解决方案提供商计划是否提供全部 Azure 服务？**

目前已推出基础结构即服务 (IaaS)、基础平台以及数据服务。 定期更新路线图，并提供类似于合作伙伴中心内的 CSP 商业文档的文档（需要进行身份验证）。 你可以在这里查看当前可用的 Azure 政府服务，但请注意，经典计算、网络、存储和云服务等部分服务不面向 CSP 提供，所以不可用。

**我需要怎么做才能借助美国政府 Microsoft 云 CSP 计划进行开发？**

有关通过美国政府 Microsoft 云合作伙伴中心进行开发的详细信息，请参阅[通过 Microsoft 政府云合作伙伴中心进行开发](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS)。

**API 是否有任何不同？**

美国政府 Microsoft 云 CSP 计划的 API 和集成要求与商业 CSP 完全相同。 有关 Microsoft Cloud for US Government 中的终结点的详细信息，请参阅[合作伙伴中心 REST URL](https://msdn.microsoft.com/library/partnercenter/mt490977.aspx)。

>[!NOTE]
>CREST API 不适用于 Microsoft Cloud for US Government 云解决方案提供商计划。 CSP 中正逐渐弃用此类 API，因此其不适用于美国政府 Microsoft 云。 为符合监管和合规要求，美国政府 Microsoft 云 CSP 计划将不提供以下 API。

**用户帐户管理** 

-   为客户创建用户帐户

-   为客户删除用户帐户

-   为客户更新用户帐户

-   获取客户所有用户帐户的列表   

-   为客户重置用户密码   

-   获取客户的用户角色    

-   设置客户的用户角色    

-   查看客户的已删除用户    

-   还原客户的已删除用户    

**许可证管理**

-   获取可用许可证的列表

-   将许可证分配给用户

-   查看为用户分配了哪些许可证

**对于不适用于 Microsoft Cloud for US Government 云解决方案提供商计划的 API，是否有变通方法？**

合作伙伴可以使用 Azure 门户、Azure AD PowerShell 或图形 API 进行合作伙伴用户帐户管理。 有关其他技术信息，请参阅[通过政府云合作伙伴中心进行开发](https://msdn.microsoft.com/partner-center/partner-center-for-microsoft-us-govt-cloud)。
