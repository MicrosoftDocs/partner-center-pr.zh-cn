## <a name="changes-to-enterprise-voice-and-audio-conferencing-extended-dial-out-launches"></a><a name="10"></a>企业语音和音频会议扩展拨出功能发布事项的变更
### <a name="categories"></a>类别

- 日期:2020-07-31
- 产品/市场

### <a name="impacted-audience"></a>影响受众

直接提供商、间接提供商和间接经销商。

### <a name="summary"></a>摘要

已取消“企业语音”和附带通话分钟数的 E5 和 A5 的发布事项。 面向美国和加拿大的扩展拨号功能发布将延迟到 2020 年 9 月 1 日。

### <a name="details"></a>详细信息

我们将取消发布此前预定于 2020 年 8 月 1 日发布的企业语音计划 1 和 2 以及 E5 和 A5 计划中随附的国内通话分钟数。 此发布计划已进入无限期搁置状态。 

面向美国和加拿大的音频会议扩展拨号功能发布将延迟到 2020 年 9 月 1 日。

#### <a name="enterprise-voice-plans-and-e5-and-a5-with-calling-minutes-launch-canceled"></a>已取消“企业语音”计划和附带通话分钟数的 E5 和 A5 的发布事项
 
**常见问题**

- **我们为什么取消发布企业语音和附带通话分钟数的 E5/A5？** 由于市场状况的不断发展和变化，我们决定取消发布这些功能。

- **这是否会影响之前已公布的电话系统、音频会议或高级通信采用产品/服务？** 不会，这些产品/服务仍将按计划提供。

## <a name="changes-coming-to-partner-center-apis"></a><a name="9"></a>即将对合作伙伴中心 API 的更改

### <a name="categories"></a>类别

- 日期:2020-07-28
- 功能

### <a name="impacted-audience"></a>影响受众

使用合作伙伴中心 API 且参与 CSP 计划的直接计费合作伙伴和间接提供商。 

### <a name="summary"></a>摘要

合作伙伴中心团队已评估了公开的合作伙伴中心 API。 我们发现，其中一些 API 已过时，不再受支持。 它们将于 2020 年 8 月 31 日停用。

### <a name="details"></a>详细信息

CSP 合作伙伴中心团队已确定了我们计划在 2020 年 8 月 31 日之前停用的一些 API。 这些 API 表示多年前设计的接口，其中许多已不再运行。 该团队正在删除 2020 年最后一个日历季度的下一个 SDK 版本中对这些终结点的引用。 以下是要停用的 API。 合作伙伴应确保以后不再引用这些 URI。

**合作伙伴服务请求** - 这些终结点使合作伙伴能够获取支持主题字符串，然后以编程方式提交合作伙伴服务请求。 随着时间的推移，此 API 变得不稳定且不受支持。 将合作伙伴定向到合作伙伴中心用户界面，以创建合作伙伴支持工单。 用户体验为合作伙伴提供了更多信息，同时创建了支持案例，例如针对他们遇到问题的部分提供建议的步骤和文档。 合作伙伴中心团队还通过自定义服务请求表单来捕获特定于问题部分的信息，从而增强了合作伙伴的工单创建体验。 联机表单收集的信息可帮助工程师更快、更准确地解决问题。 这些 URI 不应被引用或依赖，它们将于 2020 年 8 月底停用。 实现这些更改仅会影响获取支持主题和创建合作伙伴服务请求方法，所有其他与支持相关的 API 应继续按预期运行。

- {base-URL}/v1/servicerequests/supporttopics
- {base-URL}/v1/servicerequests/{agent-locale}

**Office 服务事件** - 这些终结点旨在使合作伙伴能够获取其所有合作伙伴服务中服务事件/中断的聚合视图。 但是，这些终结点返回的数据已过时且不再有用。 我们将于 7 月底停用这些终结点。 今后，合作伙伴中心团队将重新评估“所有客户”的服务事件和运行状况方案，并寻找在此领域增加价值的机会。 对这些 URI 的引用应删除。

- {base-URL}/v1/serviceincidents/office/count
- {base-URL}/v1/serviceincidents/office

合作伙伴可以使用 [Office 365 服务通信 API](https://docs.microsoft.com/office/office-365-management-api/office-365-service-communications-api-reference) 来获取客户服务的最新状态。

**TopProduct** - 创建此终结点旨在作为概念证明，从未打算将其正式发布。 它旨在使合作伙伴有一天能够管理可在合作伙伴中心目录中呈现的主要产品/服务。 但是，此 API 尚未完全开发和完成，并将于 2020 年 7 月 31 日禁用。 合作伙伴应删除对它的任何引用。

- {base-URL}/v1/preferences/catalog/products

### <a name="next-steps"></a>后续步骤

- 合作伙伴应确保其解决方案不再引用上述 API。
- 云解决方案提供商合作伙伴可以访问[合作伙伴中心 SDK 和 API Yammer 社区](https://www.yammer.com/cloudpartnercommunity/#/Threads/show?threadId=786951767490560)，询问是否需要进一步说明。

## <a name="sdk-release-on-net-standard-v1161-preview"></a><a name="8"></a>.NET Standard 上的 SDK 版本 (v1.16.1-preview)

### <a name="categories"></a>类别

- 日期:2020-07-21
- 功能

### <a name="impacted-audience"></a>影响受众

使用合作伙伴中心 .NET SDK 且参与 CSP 计划的直接计费合作伙伴和间接提供商。 

### <a name="summary"></a>摘要

从 2020 年 7 月 21 日开始，合作伙伴可以开始下载预览版 .NET Standard SDK Microsoft 合作伙伴中心 .NET SDK v1.16.1-preview，以及更新的公共合作伙伴中心 SDK GitHub 示例 。

### <a name="details"></a>详细信息

参与 CSP 计划的直接计费合作伙伴和间接提供商都有资格下载 .NET standard SDK 的最新预览版。 我们会将 Microsoft 合作伙伴中心 SDK 从 .NET Framework 迁移到 .NET Standard 2.0 平台。 这些更改可以使 SDK 与使用 .NET Framework 4.6.1 及更高版本的现有应用程序兼容。 .NET Standard 将支持 .NET Core 2.0 及更高版本。

合作伙伴请求 .NET Standard 中的 SDK 的需求非常大。 当前版本的 SDK 强制合作伙伴使用以 ASP.NET Framework 编写的应用程序，该应用程序比 .NET Core 应用程序更慢且更贵。 因此，迁移到 .NET standard 可帮助解决性能或与 .NET Core 兼容的问题。
 
**要点**
- Microsoft 不再针对 .NET Framework 4.8 进行更多发布，因为它是所在系列中的最后一个版本。
- 限制未来的开发，因为大多数库将针对 .NET Core 或 .NET standard 进行发布，而后者可能不支持当前的框架版本。
- .NET Core 具有轻量特性，因此非常适合与容器一起使用。
- Windows Server 和 Linux 的高性能服务器运行时使 .NET Core 成为性能最佳的 Web 框架。

### <a name="next-steps"></a>后续步骤

- 下载最新版本的 [.NET SDK v1.16.1-preview](https://www.nuget.org/packages/Microsoft.Store.PartnerCenter/1.16.1-preview)
- 版本：.NET SDK v1.16.1-preview 
- 示例代码：[示例代码](https://github.com/microsoft/Partner-Center-DotNet-Samples)
- 预览版可用日期：2020/7/21 
- 正式发布：预览版发行后的三周（为合作伙伴留出足够的迁移时间） 
- 当前版本中没有新的 API 更改

## <a name="cloud-solution-provider-csp-incentive-for-eligible-microsoft-commercial-marketplace-offers-enhanced-for-fy21"></a><a name="7"></a>针对 2021 财年加强的符合条件的 Microsoft 商业市场产品/服务的云解决方案提供商 (CSP) 奖励

### <a name="categories"></a>类别

- 日期:2020-07-22
- 功能

### <a name="impacted-audience"></a>影响受众

参与 CSP 计划的直接计费合作伙伴和间接提供商

### <a name="summary"></a>摘要

从 2020 年 7 月 1 日开始，CSP 直接计费合作伙伴和间接提供商可以在交易符合条件的基础结构即服务 (IaaS) 产品/服务时获得奖励。

### <a name="details"></a>详细信息

参与 CSP 计划的直接计费合作伙伴和间接提供商有资格从 [Microsoft 商业市场](https://azure.microsoft.com/overview/commercial-marketplace)交易其客户的精选第三方产品/服务。 作为增加商业市场收入的一种回报，符合条件的 CSP 直接计费合作伙伴和间接提供商在交易符合条件的产品/服务时可获取奖励。

从 2020 年 7 月 1 日开始，在交易符合条件的 IaaS 产品/服务时可获取奖励。 针对符合条件的软件即服务 (SaaS) 产品/服务的奖励将在 2021 财年继续进行。

在 [Microsoft 合作伙伴网站](https://aka.ms/partnerincentives)上可以找到符合奖励条件的 SaaS 和 IaaS 产品/服务的最新列表。 符合条件的产品/服务列表每月更新一次，并发布到 CSP Direct Bill Partner 奖励计划和 CSP Indirect Provider 奖励计划的资产集合中。 CSP 合作伙伴可以通过合作伙伴中心或 Azure 门户启动这些交易。

要获取奖励，需要满足相关奖励计划指南中所述的所有资格和注册要求。 [合作伙伴网站](https://aka.ms/partnerincentives)上提供了计划指南和其他有用的奖励资源。

### <a name="next-steps"></a>后续步骤

- 请参阅完整公告，并在 Operations Readiness 资源库集合：[针对符合条件的 Microsoft 商业市场产品/服务的 CSP 奖励](https://partner.microsoft.com/resources/collection/csp-incentive-for-microsoft-commercial-marketplace-offers#/)中了解有关商业市场的详细信息。
 
- 在每月初查看符合条件的 SaaS 和 IaaS 产品/服务的列表。
 
- 在 CSP 计费关系中鼓励客户提供这些服务。

## <a name="new-capabilities-added-to-the-insights-reports-in-partner-center"></a><a name="6"></a>合作伙伴中心的“见解”报表中添加了新功能 


### <a name="categories"></a>类别

- 日期:2020-07-16
- 功能
 
 
### <a name="impacted-audience"></a>影响受众

Microsoft 合作伙伴网络中的所有合作伙伴组织 

### <a name="summary"></a>摘要

合作伙伴中心的“见解”报表中添加了新功能，合作伙伴可根据业务需求在该报表中执行临时数据分析，并查看组织内部员工所接受培训的整体情况。  这些新功能扩充了[现有“见解”报表](../partner-center-insights.md)的稳健集合。

新功能概述：

- **原始数据导出**，支持客户详细信息、订阅详细信息、Microsoft Azure 使用情况、Office 365 使用情况、Dynamics 365 使用情况、企业移动性和安全性 (EMS) 使用情况、Power BI 使用情况、培训和认证以及 Microsoft Learn 等方面的原始数据导出。 有关详细信息，请参阅[此文章](../pci-download-reports.md)。

- **新的培训仪表板**，提供组织内部员工所接受培训的整体情况。 有关详细信息，请参阅[此文章](../pci-training-dashboard.md)。

### <a name="what-is-insights"></a>“见解”仪表板的作用是什么？

通过合作伙伴中心的“见解”仪表板，你可以跨云产品（如 Office 365、Azure、Dynamics 365）和许可模式（如云解决方案提供商 (CSP) 和企业协议 (EA)）获取合作伙伴组织关键绩效指标 (KPI) 的 360 度统一视图。 还可获取智能见解，清楚地了解与 Microsoft 的业务，并满怀信心地做出数据驱动的决策。   

要查看此仪表板，请在合作伙伴中心的左侧导航菜单中选择“见解”。 此仪表板使用与当前 MyInsights 报表工具相同的基础数据，但提供了更丰富的视觉对象，以及更具洞察力的报表功能和 KPI，这些功能和 KPI 涵盖客户、订阅、许可证使用情况、胜任度和权益。 请经常返回查看，因为我们会继续在“见解”仪表板中引入新的报表和功能。    

### <a name="next-steps"></a>后续步骤

我们鼓励你探索这些新功能，并使用“见解”选项卡中提供的内嵌反馈工具分享你的反馈。

## <a name="perpetual-software-in-the-cloud-solution-provider-program"></a><a name="5"></a>云解决方案提供商计划中的永久软件 


### <a name="categories"></a>类别

- 日期:2020-07-15

### <a name="impacted-audience"></a>影响受众

CSP 间接经销商以及符合条件的 CSP 间接提供商 

### <a name="details"></a>详细信息

在 Microsoft，我们致力于帮助每个组织完成数字化转型。 为此，我们不仅投资技术，还研究从获取技术到提供支持的端到端客户和合作伙伴历程。 通过这项工作，我们发现了实现自我转型的机会，即投资我们的商业平台。  我们还在不断简化许可计划，同时发展新的商业体验，以便改善为合作伙伴提供更多机会的方式。 

随着我们改进商业功能以便为客户和合作伙伴提供更好的购买体验，我们持续扩展产品组合，实现在云解决方案提供商 (CSP) 计划中提供永久软件许可证。 在 CSP 中添加软件许可证后，参与的合作伙伴便可以专注于向其客户提供各种解决方案和托管服务。 它还使客户能够过渡到云，并为合作伙伴提供混合云环境所需的灵活性。   

从 7 月 1 日开始，我们将通过一组符合条件的间接提供商及其间接经销商来增加此功能的采用，这些提供商和经销商将能够与 CSP 计划中的客户交易本地软件的许可证。 符合条件的合作伙伴由 Microsoft 管理，并且必须既是活跃的 CSP 间接提供商，又是开放许可计划的 Microsoft 授权分销商，该计划在当前支持在 CSP 中销售软件的地域运营。 从 2021 年 1 月开始，Microsoft 将进一步扩展此功能，使所有 CSP 直接账单合作伙伴、间接提供商和间接经销商都可以在 CSP 计划中销售软件许可证。
