## <a name="microsoft-will-rename-office-365-small-and-medium-business-smb-offers-from-april-21-2020"></a><a id="7"/></a>从 2020 年 4 月 21 日起，Microsoft 将对 Office 365 中小型企业 (SMB) 产品/服务进行重命名

### <a name="categories"></a>类别

- 日期:2020-03-30
- 键入：产品/市场
- 状态：即将推出

### <a name="summary"></a>摘要：

2020 年 4 月 21 日，Microsoft 将重命名所有 Office 365 SMB 产品/服务。 这是一次以更一致的方式向我们的客户传递我们的价值和产品系列的绝佳机会。

### <a name="impacted-audience"></a>影响受众：

所有云解决方案提供商 (CSP) 合作伙伴。

### <a name="details"></a>详细信息

2020 年 4 月 21 日，我们将对我们 Office 365 SMB 产品/服务的名称进行更改：

- Office 365 商业协作版将更名为 Microsoft 365 商业基础版。 这是我们的入门级产品/服务，适用于寻求基础云服务的客户。
- Office 365 商业高级版将更名为 Microsoft 365 商业标准版。 这是我们面向 SMB 的核心产品/服务，包含了云服务和连接云的桌面应用。
- Microsoft 365 商业版将更名为 Microsoft 365 商业高级版。 这仍是我们面向寻求额外的安全性、设备和标识价值的高端 SMB 首推的 SKU。

我们还将采用 Microsoft 365 应用版作为我们两个 Office 客户端订阅的新名称：

- Office 365 商业版将更名为 Microsoft 365 商业应用版
- Office 365 专业增强版将更名为 Microsoft 365 企业应用版

### <a name="impact-on-your-operations"></a>对你的运营造成的影响

价格、功能或业务模型不会有任何更改。 此外，Office 365 企业版 SKU 和 Microsoft 365 企业版 SKU 的命名也没有任何更改。 所有 SKU 和产品/服务的重命名将自动流入合作伙伴中心中的订阅名称。

### <a name="partner-opportunity"></a>合作伙伴机会

这次的重命名使客户能够更好地了解每项产品/服务所含的增量价值，使你能够更好地针对你的帐户定位追加销售的机会。

此外，它还解决了 Office 365/Microsoft 365 商业版计划中名称不一致的问题。 同样，这次的重命名会清楚地将通过新的 Microsoft 365 商业/企业应用版销售的核心 Office 应用程序从范围更广的计划中分隔出来。

### <a name="additional-resources"></a>其他资源

请访问我们的 [M365 合作伙伴登陆页面](https://www.microsoft.com/microsoft-365/partners/smb-sku-rename)以了解更多信息。

### <a name="next-steps"></a>后续步骤

请熟悉价目表的内容，并与你组织中所有合适的联系人共享此信息。<br>

特别要留意你的销售运营和市场营销同事，他们可能需要对产品宣传材料进行更新以反映出这些新名称。

**有疑问？**

有关这些产品/服务的任何其他问题，请查看相关的 Yammer 社区并联系你的 Microsoft 客户团队代表。 让我们一起完成出色的工作。

## <a name="new-partner-center-capabilities-to-help-direct-bill-partners-complete-their-transition-to-indirect-reseller"></a><a id="6"/></a>新的合作伙伴中心功能，用于帮助直接计费合作伙伴完成到间接经销商的转换

### <a name="categories"></a>类别

- 日期:2020-03-20
- 键入：功能
- 状态：现已推出
- [了解详细信息](https://docs.microsoft.com/partner-center/)

### <a name="summary"></a>摘要：

我们提供了新的功能，帮助客户将他们的 CSP 订阅从直接计费租户转换到间接提供商租户。

### <a name="impacted-audience"></a>影响受众：

要转换到间接经销商和间接提供商的直接计费合作伙伴。

### <a name="details"></a>详细信息

从 2018 年 8 月 31 日起，Microsoft 已经更新了针对云解决方案提供商 (CSP) 计划中直接计费合作伙伴的注册要求。 随着云解决方案提供商计划的不断发展，我们希望确保所有合作伙伴都能相应地满足不断增长的客户需求。

为此，我们提供了将客户的 CSP 订阅从直接计费租户转换到间接提供商租户的新功能。 这一新功能将简化直接计费合作伙伴转换到间接经销商的这一过程，有助于加速计划内的企业发展。

有关详细信息，请参阅[云解决方案提供商产品/服务](https://docs.microsoft.com/partner-center/csp-offers)页面。

注意：转移仅支持基于许可证的订阅和 Azure 订阅。 你需要手动转移 Azure 计划和商业市场购买内容。 请注意，这涉及到对以下文档中所述的订阅进行取消和续订操作：[Azure 计划](https://docs.microsoft.com/partner-center/azure-plan-lp)和[商业市场](https://docs.microsoft.com/partner-center/csp-commercial-marketplace-manage)。 由于服务器订阅和 Azure 预留是一次性购买内容，因此不需要将它们转移到间接提供商租户。 有关详细信息，请参阅“云解决方案提供商产品/服务”页面。

### <a name="additional-resources"></a>其他资源：

- [从云解决方案提供商 (CSP) 计划直接计费合作伙伴过渡到云解决方案提供商计划间接经销商](https://docs.microsoft.com/partner-center/transition-direct-to-indirect#move-your-direct-bill-customers-to-your-indirect-provider)帮助文章
- [直接计费合作伙伴到间接经销商](https://assetsprod.microsoft.com/mpn/direct-bill-transition-to-indirect-reseller-new-capability)

## <a name="updated-changes-to-education-customer-validation-in-csp"></a><a id="4"/></a>已更新：对云解决方案提供商中教育版客户验证的更改

### <a name="categories"></a>类别

- 日期:2020-03-10
- 键入：功能
- 状态：现已推出
- [了解详细信息](https://partner.microsoft.com/resources/collection/csp-aer-partner-qualification-retirement#/)

### <a name="impacted-audience"></a>影响受众：

从事教育/学术产品/服务交易的直接计费和间接提供商 CSP 合作伙伴

### <a name="summary"></a>摘要：

新功能通知：合作伙伴现在可以更正客户帐户信息并重新提交，以用于对客户被拒案例的审核。 有关完整详细信息，请参阅已更新的[“在合作伙伴中心中创建教育客户”指南](https://assetsprod.microsoft.com/mpn/partner-center-education-customer-creation.pdf)，了解如何执行此操作的分步说明。

-  要减少对合法条目的审核失败的调查延迟，请确保输入准确的客户数据，并尽可能完整。 不要输入测试/虚拟数据。
-  查看“在合作伙伴中心创建教育客户”指南和“添加新客户”页面，了解在合作伙伴中心中创建新教育客户所需的内容。 下面的资源库中提供了这两个资源。

**注意：** 现在，“在合作伙伴中心创建教育客户”指南包括关于如何更正客户信息以及在客户审核失败的情况下重新提交信息的分步说明。

### <a name="details"></a>详细信息

Microsoft 正在移除 Microsoft Defender 高级威胁防护 (MDATP) 中的 Windows E3 许可证先决条件。 CSP 合作伙伴将能够为其所有受支持的客户端设备购买 MDATP，无需考虑 Windows E3 许可证的所有权。

为实现此业务模型，Microsoft 已推出了面向 CSP 的独立产品/服务，这些产品/服务现已可用。

此外，现有的 Windows E5、Microsoft 365 E5 安全性和 Microsoft 365 E5 不会有任何更改，这些产品/服务将继续包含 MDATP。
