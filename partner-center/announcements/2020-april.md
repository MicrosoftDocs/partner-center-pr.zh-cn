## <a name="updated-promotions-for-office-365"></a><a id="14"/></a>更新了 Office 365 的促销

### <a name="categories"></a>类别

- 日期:2020-04-30
- 键入：功能
- 状态：即将推出
- [了解详细信息](https://aka.ms/CSPCOVIDPromo)

### <a name="summary"></a>摘要

2020 年 5 月 4 日（UTC 凌晨 12:01），Microsoft 将针对Office 365 E1 和 Microsoft 365 商业基础版推出新的促销活动，并替代 Office 365 E1 当前为期 6 个月的试用版（针对新客户）。

### <a name="impacted-audience"></a>影响受众

参与云解决方案提供商 (CSP) 计划的所有合作伙伴。 本次促销仅面向新商业客户。

### <a name="details"></a>详细信息

在这些前所未有的时期，[Microsoft 对客户的承诺](https://www.microsoft.com/microsoft-365/blog/2020/03/05/our-commitment-to-customers-during-covid-19/)包含了这样一部分内容：在 CSP 中为新客户提供为期 6 个月的 Office 365 E1 试用版（请参阅下面的定义）。
2020 年 5 月 4 日，Microsoft 将为 Office 365 E1 和 Microsoft 365 商业基础版引入新的折扣促销，以在按月付费的年度订阅前 6 个月免费为新客户提供这些产品。

这些促销将取代当前的 Office 365 E1 试用版产品/服务，并在以下方面有所不同：

- 合作伙伴将利用 CSP 促销来对年度 Office 365 E1 或 Microsoft 365 商业基础版 SKU 进行交易，该促销在前 6 个月提供 100% 的折扣，而不是为期 6 个月的唯一的试用 SKU。 [了解有关 CSP 促销的详细信息](https://docs.microsoft.com/partner-center/promotions)。
- 订购将严格限制为“仅限新客户”，而不是 Office 365 E1 试用版（仅通过书面策略来限制为新客户）。 查看 [Microsoft 合作伙伴常见问题解答](https://aka.ms/CSPCOVIDPromo)，了解详细信息。
- 对于 SMB 客户，本次促销中将包含 Microsoft 365 商业基础版。 这可确保合作伙伴能够使用 Microsoft 365 商业版帮助席位不到 300 的客户使用适当的产品系列。
当前获得 Office 365 E1 试用版授权（2020 年 5 月 4 日前可用）的客户仍将处于其当前计划中；此更改仅会影响新客户。

新客户的定义

符合本次促销条件的新客户包括：

1. 全新客户租户
2. 符合以下条件的现有客户：对于 [Microsoft 合作伙伴常见问题解答 (FAQ) 指南](https://aka.ms/CSPCOVIDPromo)中定义的任何产品，其租户中没有当前已付费或最近取消（最近 30 天之内）的订阅。

### <a name="additional-resources"></a>其他资源

- 访问[合作伙伴远程办公资源中心](https://www.microsoft.com/microsoft-365/partners/remotework)，了解有关安全远程办公的详细信息。
- 有关这些更改的详细信息，请参阅[相关合作伙伴常见问题解答](https://aka.ms/CSPCOVIDPromo)。

## <a name="azure-reservations-new-customer-self-serve-capability-with-partner-permission"></a><a id="13"/></a>“Azure 预留”的新客户自助服务功能（具有合作伙伴权限）

### <a name="categories"></a>类别

- 日期:2020-04-29
- 键入：功能
- 状态：现已推出
- [了解详细信息](https://partner.microsoft.com/resources/collection/customer-permission-to-purchase#/)

### <a name="impacted-audience"></a>影响受众

云解决方案提供商计划中的合作伙伴

### <a name="summary"></a>摘要

Microsoft 致力于提供灵活的选项，方便客户从 Microsoft 和我们的合作伙伴处购买和使用产品及服务。

为了持续践行我们的承诺，我们针对 Azure 预订引入了一项新的客户自助服务功能，该功能简化了客户购买体验，并通过 CSP 中的合作伙伴进一步提高了 Azure 预订的使用率。

Azure 预订是以折扣价购买 Azure 服务的一种绝佳方式。 利用此新功能，合作伙伴可以选择为客户授予相应权限，让客户可以在其已经为客户购买的现有 Azure订阅中灵活地购买自己的资源。

以下是有关该功能工作方式的一些详细信息：

- 合作伙伴可以选择在单个客户帐户级别启用该功能，并向客户提供购买控制权。
- 默认情况下，此功能设置为“关”，合作伙伴可选择启用并授予客户权限。
- 合作伙伴可在合作伙伴中心内验证和管理客户的购买。
- 可使用 API 大规模启用客户权限，以自动执行相关过程。

### <a name="next-steps"></a>后续步骤

1. 开始帮助客户使用新功能购买自己的 Azure 预订。
2. [查看此集合中的可用资源](https://partner.microsoft.com/resources/collection/customer-permission-to-purchase#/)，其中提供了相关详细信息，包括[技术文档](https://docs.microsoft.com/partner-center/give-customers-permission)、合作伙伴体验浏览和客户体验浏览。

## <a name="microsoft-reversed-the-csp-eur-price-for-microsoft-365-e3"></a><a id="12"/></a>Microsoft 恢复了 Microsoft 365 E3 的云解决方案提供商欧元价格。

### <a name="categories"></a>类别

- 日期:2020-04-24
- 键入：产品/市场
- 状态：现已推出
- [了解详细信息](https://flow.microsoft.com/ui-flows/)

### <a name="summary"></a>摘要

Microsoft 在 2020 年 4 月 1 日修复了 Microsoft 365 (M365) E3 的欧元定价错误。 但在 2020 年 4 月 22 日，Microsoft 暂时恢复了该价格，以帮助合作伙伴应对当前的特殊情况。

### <a name="impacted-audience"></a>影响受众

在 2020 年初，我们发现 CSP 中的 Microsoft 365 E3 欧元定价存在错误：净价低于应有的价格。 我们安排了系统更新来纠正该价格，在 2020 年 4 月 1 日修正了该价格。

虽然之前的定价出现错误，但在当前的特殊情况下并不适宜进行修正和提高价格。 因此，我们将暂时恢复之前更低的净价，于 2020 年 4 月 22 日生效。 所有发票将立即反映以前的较低价格，而在可预见的将来，价目表将显示 2020 年 5 月价目表中的较低价格。

合作伙伴不应使用 2020 年 4 月价目表上列出的 M365 E3 欧元定价；而是应参考 2020 年 3 月价目表上列出的价格。

非常感谢合作伙伴社区在这个前所未有的时期完成的所有出色工作。

## <a name="reminder-activation-of-security-safeguards-for-partner-center-transactions-in-csp"></a><a id="11"/></a>提醒：为云解决方案提供商计划中的合作伙伴中心事务激活安全保护措施

### <a name="categories"></a>类别

- 日期:2020-04-23
- 键入：功能
- 状态：即将推出
- [了解详细信息](https://partner.microsoft.com/resources/collection/partner-security-requirements#/)

### <a name="target-partner-audiences"></a>面向合作伙伴受众

- 参与云解决方案提供商 (CSP) 计划的所有合作伙伴
- 所有控制面板供应商 (CPV)
- 所有 Azure 顾问合作伙伴

### <a name="summary"></a>摘要

提供更大力度的持续性安全和隐私保护是我们的首要任务，并且我们会继续帮助合作伙伴保护他们的客户和租户。 在 2019 年，我们引入了[强制性安全要求](https://blogs.partner.microsoft.com/mpn/safeguard-business-security-best-practices/)。 所有加入云解决方案提供商计划、控制面板供应商计划和顾问计划的合作伙伴都需要实施该要求以保持合规性。

最近，我们已成功为所有合作伙伴租户完成了针对“管理员代表”(AOBO) 功能的安全保护措施激活操作，旨在保护其业务和客户免受身份盗窃相关事件的侵害。

为了进一步帮助合作伙伴和客户，从 2020 年 5 月 1 日起，我们将开始为云解决方案提供商计划中的合作伙伴中心交易激活安全措施。 激活附加的安全保护措施意味着，合作伙伴需要使用多重身份验证 (MFA) 才能在云解决方案提供商计划中完成涉及合作伙伴中心用户体验或 API 的事务，其目的是防止未经授权的访问以及保护其客户和租户免受与身份盗窃相关的安全事件的侵害。 请访问[此处](https://docs.microsoft.com/partner-center/partner-security-requirements-mandating-mfa)了解更多详细信息。

### <a name="next-steps"></a>后续步骤

- 如果尚未激活建议的安全要求来保护租户和用户，请立即执行此操作以保持合规性，并避免在云解决方案提供商计划中管理客户或处理事务时发生业务中断。 不实施安全要求的合作伙伴在参与云解决方案提供商计划时将面临安全风险。
- 间接提供商必须为其自己的公司实施这些安全要求，并与经销商合作，确保他们也使用此电子邮件目标实施了这些要求。 我们还会通过电子邮件直接与经销商联系。
- 请访问[合作伙伴中心安全要求状态页](https://partner.microsoft.com/pcv/security/compliance)，确认自己的进度和要采取的关键操作。 与全局管理员协作，为每个租户定期检查安全要求的进度。
- 如果正在使用控制面板供应商 (CPV) 解决方案，请咨询控制面板供应商。

### <a name="additional-resources"></a>其他资源

查看这些资源以帮助实施安全要求。

- 查看用于实施合作伙伴安全要求的[分步指南](https://docs.microsoft.com/partner-center/partner-security-requirements)。
- 参阅有关合作伙伴安全要求的[常见问题](https://docs.microsoft.com/partner-center/partner-security-requirements-faq)和解答。
- 查看[安全要求状态检查指南](https://docs.microsoft.com/partner-center/partner-security-compliance)。
- 在[实施合作伙伴安全要求](https://partner.microsoft.com/resources/collection/partner-security-requirements#/)集合中访问其他资源。
- 将[合作伙伴中心安全指导社区小组](https://aka.ms/MPCSecurityGuidance)加入书签。

如何更多有关技术问题的帮助，请[提交支持票证](https://partner.microsoft.com/dashboard/support/csp/servicerequests/create?topicid=32674991)。 请将此电子邮件转发给你组织中其他可能需要采取措施来实施这些安全要求的人员。

感谢你对你的合作伙伴和对确保生态系统以信任为基础运行所做的努力。

注意：如果正在使用基线策略，且尚未转换到安全默认策略或[其他 MFA 实现选项](https://docs.microsoft.com/partner-center/partner-security-requirements#actions-that-you-need-to-take)，请尽快参考[此处的关键资源](https://partner.microsoft.com/resources/collection/security-defaults-enable-mfa#/)实现转换。

## <a name="announcing-microsoft-defender-advanced-threat-protection-mdatp-for-servers-offer-available-for-csp-on-april-1-2020"></a><a id="9"/></a>宣布于 2020 年 4 月 1 日面向云解决方案提供商推出“适用于服务器的 Microsoft Defender 高级威胁防护 (MDATP)”这一产品/服务。

### <a name="categories"></a>类别

- 日期:2020-04-17
- 键入：产品/市场
- 状态：现已推出

### <a name="summary"></a>摘要

“适用于服务器的 Microsoft Defender ATP”SKU 将于 2020 年 4 月 1 日面向 CSP 推出，定价为 5 美元/OSE/月。

### <a name="impacted-audience"></a>影响受众

这个新的 SKU 适用于至少拥有 50 个许可证（MDATP、Win E5、M365 安全性、M365 E5）的 Microsoft Defender 高级威胁防护 (ATP) 客户，这些客户希望通过 Microsoft Defender ATP 实现服务器终结点保护。

### <a name="details"></a>详细信息

Microsoft 针对服务器（本地服务器和 Azure 中的服务器）上的终结点保护所提供的产品/服务为 Azure 安全中心标准版。 Azure 安全中心标准版的成本为每月每服务器操作系统环境 15 美元。 我们从客户那里得知，他们希望能够选择价位更低的功能缩减版，可以采用与购买 E5 一致的方式来购买以实现终结点服务器保护。
为了响应客户的号召，我们引入了新的“适用于服务器的 Microsoft Defender ATP”SKU，以此来满足客户的需求，并且我们希望大多数客户能够将此 SKU 用于本地服务器，但它并不局限于该用例。 Azure 安全中心标准版仍然是具有一组扩展的威胁防护功能的首推 SKU，合作伙伴应向客户介绍 Azure 安全中心标准版首推 SKU，特别是针对使用 Azure 工作负荷的客户。 不过，现在 Microsoft 可以以更低的价位提供终结点保护，这样一来，使 Microsoft 合作伙伴能够凭借“适用于服务器的 Microsoft Defender ATP”SKU 在终结点保护市场上更具竞争性。

至少拥有 50 个许可证的 Microsoft Defender ATP 客户可以使用新的“适用于服务器的 Microsoft Defender ATP”SKU 实现终结点保护，费用为每月每服务器操作系统环境 5 美元。 若要为服务器实现增强的云工作负荷保护，“适用于服务器的 Microsoft Defender ATP”SKU 客户可以升级到 Azure 安全中心标准版 SKU，只需每月为每服务器 OSE 额外花费 10 美元即可享受自适应应用程序控制、实时虚拟机访问、文件完整性监测等功能，并获得保护 Azure 资源和其他价值的统一体验（例如未来与 Azure Arc 的集成）。

目前 Azure 安全中心标准版没有任何更改。 Azure 安全中心标准版会继续提供集成的用户体验，不仅可以保护服务器虚拟机终结点，还提供了一系列 Azure 资源，包括 IoT、Azure Kubernetes 服务（预览版）、Azure 容器注册表（预览版）、Key Vault（预览版）、应用服务、SQL 以及存储。

“适用于服务器的 MDATP”SKU 所有者需要通过 10 美元的升级 SKU 升级到 Azure 安全中心 (ASC) 标准版，以此获得 ASC 标准版功能。

## <a name="microsoft-365-business-voice-available-to-csp-partners-from-april-1-2020"></a><a id="2"/></a>Microsoft 365 商务语音从 2020 年 4 月 1 日起向云解决方案提供商合作伙伴推出

### <a name="categories"></a>类别

- 日期:2020-04-01
- 键入：产品/市场
- 状态：现已推出
- [了解详细信息](https://partner.microsoft.com/resources/collection/microsoft_365_business_voice_in_csp#/)

### <a name="summary"></a>摘要

Microsoft 365 商务语音是一个基于云的电话系统，专为中小型企业所打造，将从 2020 年 4 月 1 日起在此处列出的国家/地区推出。

### <a name="impacted-audience"></a>影响受众

- 在这些[国家/地区](https://partner.microsoft.com/resources/collection/microsoft_365_business_voice_in_csp#/)运营的所有 CSP 合作伙伴。
- 澳大利亚的合作伙伴须查看我们下面的指南。

### <a name="details"></a>详细信息

2020 年 4 月 1 日，Microsoft 将向其他地区引入 Microsoft 365 商务语音。 我们将推出以下两项产品/服务：Microsoft 365 商务语音，以及 Microsoft 365 商务语音（不含通话套餐），后者使用的是直接路由。 这两项产品/服务都将在美国推出。

### <a name="customer-value-proposition"></a>客户价值主张

Microsoft 365 商务语音是基于云的电话系统，适用于多达 300 个用户的企业。 它使用户可在 Microsoft Teams 中拨打和接听任何电话。 通过与 Teams 配合使用，它可以提供将通话、聊天和会议结合在一起的一体式通信解决方案。

Microsoft 365 商务语音对高效工作和通信功能进行了简化和优化，将它们集成到了单一应用程序中，可随时随地通过任何设备处理业务：无论是计算机、手机，还是桌面电话。 Microsoft 365 商务语音的优点包括：

- 一体式通信。 通过统一通话、聊天、会议、日历和电子邮件功能来转换完成工作的方式。
- 轻松且智能的通话。 实现更灵活、更轻松以及更智能的通话，使你随时保持连接。
- 通过 Microsoft 云提供。 凭借可靠、安全的基于云的解决方案自信地工作。

Microsoft 365 商务语音将作为以下计划的加载项提供：

- Office 365 商业协作版、商业高级版、F1、A1、E1、A3 和 E3
- Microsoft 365 商业版、F1、A3 和 E3

### <a name="partner-value-proposition"></a>合作伙伴价值主张

合作伙伴可以通过以下方式利用此机会实现盈利：

- 对带有呼叫队列/自动助理的电话系统提供支持、评估网络质量，或者配置直接路由和音频会议。
- 提供用户采用/培训服务，从而确保组织已准备就绪并开始使用现代通信工具 Microsoft Teams。
- 通过质量报告、设备管理和用户支持等托管服务支持并监视客户环境。

**Microsoft 365 商务语音（美国、加拿大、英国）**

继我们于 2019 年 11 月在英国和加拿大推出了商务语音（含通话套餐）后，我们将于 2020 年 4 月 1 日向美国客户推出此产品/服务，供他们购买。

对于需要国际呼叫功能的客户，还可购买附加产品/服务。

- Microsoft 365 商务语音：9f9f2c7b-c961-402b-9421-8e3c9207eeb3
- 适用于 SMB 的 Microsoft 365 国际通话套餐：c01a89e5-d246-42d6-b21a-f4a1e9b04f56

**Microsoft 365 商务语音（不含通话套餐）（70 多个国家/地区）**

或者，从 2020 年 4 月 1 日起，可以购买商务语音（不含通话套餐）这一产品/服务，其中包含了电话系统和音频会议。 此产品/服务将在此处列出的所有国家/地区推出。

- **仅限美国：** Microsoft 365 商务语音（不含通话套餐）：39d77d0f-eb8f-4ebc-b618-692e1cc59c8f
- **世界其他国家/地区：** Microsoft 365 商务语音（不含通话套餐）：cc69a07c-8c51-457f-bb2a-f21a62d6bede

请与我们位于你当地的子公司联系以了解如何通过此产品/服务获益。

### <a name="additional-guidance-for-partners-in-the-united-states-and-puerto-rico"></a>适用于美国和波多黎各合作伙伴的其他指导信息

在美国和波多黎各，商务语音是一项含税服务。 Microsoft 向合作伙伴收取的费用以及合作伙伴向 Microsoft 支付的费用包括适用的美国电信税费。 在其他市场中，则按普通开票流程收取税金。 在决定提升商务语音价格之前，请就缴税事宜咨询法律顾问。 如果合作伙伴提升了商务语音的价格，则需要通过某种方式来缴纳电信税。 否则，合作伙伴将不会缴纳溢价部分的增量税金，从而导致违反纳税规定。 Microsoft 为美国的商务语音合作伙伴提供额外的云解决方案提供商奖励，以补偿可能缺少的利润。

### <a name="guidance-for-partners-in-australia"></a>针对澳大利亚合作伙伴的指南

澳大利亚的合作伙伴须留意，Microsoft 365 商务语音（不含通话套餐）这项产品/服务将不会提供在 2020 年 4 月 1 日的产品价目表上 – 尽管这项产品/服务已在 2020 年 3 月的预览价目表上提供。 感谢你的理解。

## <a name="act-now-partners-are-now-required-to-have-accepted-the-microsoft-partner-agreement"></a><a id="1"/></a>立即行动：合作伙伴现在需要接受 Microsoft 合作伙伴协议。

### <a name="categories"></a>类别

- 日期:2020-04-01
- 键入：功能
- 状态：现已推出
- [了解详细信息](https://partner.microsoft.com/resources/collection/new-microsoft-cloud-agreement-csp)

### <a name="impacted-audience"></a>影响受众
所有云解决方案提供商计划合作伙伴。

**2020 年 4 月 1 日更新**截至 2020 年 1 月 31 日，云解决方案提供商 (CSP) 计划中的所有合作伙伴都应接受针对他们所有交易 CSP 租户的协议。 之前，我们传递的信息是协议的下一个实施阶段将从 2020 年 4 月开始。  然而，为了防止对业务造成任何中断，我们已决定将下一个阶段的实施延长至 2020 年 4 月之后进行。 我们收到了来自一些云解决方案提供商合作伙伴的反馈，他们在接受 MPA 时遇到了问题。 基于这些情况，我们正努力保障 Microsoft 资源已做好为这些接受协议时遇到困难的合作伙伴提供支持的准备。

### <a name="summary"></a>摘要
截至 2020 年 1 月 31 日，云解决方案提供商计划中的所有合作伙伴（包括间接经销商）都应接受 Microsoft 合作伙伴协议，从而确保能够继续开发新客户或为 CSP 计划中的现有客户新下订单。 CSP 计划中的所有合作伙伴现在必须接受 Microsoft 合作伙伴协议。

自 2020 年 4 月起：
- 尚未接受 Microsoft 合作伙伴协议的直接计费合作伙伴和间接提供商将无法新下订单，并且仅能管理现有订阅。
- 尚未接受 Microsoft 合作伙伴协议的间接经销商将不再有资格担任新订单的记录合作伙伴。

如果尚未执行此操作，请立即完成以下操作。

### <a name="actions-for-direct-bill-partners"></a>所有直接计费合作伙伴需要执行的操作：
1. 通知组织中的主要利益干系人查看 Microsoft 合作伙伴协议。
2. 使用全局管理员凭据在[合作伙伴中心的云解决方案提供商概述页面](https://partner.microsoft.com/pcv/dashboard/overview)上接受 Microsoft 合作伙伴协议。

可以通过[此处](https://partner.microsoft.com/resources/collection/new-microsoft-cloud-agreement-csp#/)的 CSP 库页面获取更多信息和资源。

### <a name="actions-for-indirect-providers"></a>间接提供商需要执行的操作
1. 通知组织中的主要利益干系人查看 Microsoft 合作伙伴协议。
2. 使用全局管理员凭据在[合作伙伴中心的云解决方案提供商概述页面](https://partner.microsoft.com/pcv/dashboard/overview)上接受 Microsoft 合作伙伴协议。
3. 使用 [API 来验证你的间接经销商的签署状态](https://docs.microsoft.com/partner-center/develop/verify-indirect-reseller-mpa-status)。
4. 帮助你的间接经销商完成以下操作。

可以通过[此处](https://partner.microsoft.com/resources/collection/new-microsoft-cloud-agreement-csp#/)的 CSP 库页面获取更多信息和资源。

我们了解到了间接提供商在测试交易执行功能时遇到的技术问题。 这些问题将于 2020 年 4 月前解决。

### <a name="actions-for-indirect-resellers"></a>间接经销商需要执行的操作
1. 以间接经销商的身份[加入合作伙伴中心](https://partner.microsoft.com/resources/collection/indirect-reseller-onboarding-for-mpa-in-csp#/)，并在加入过程中接受 Microsoft 合作伙伴协议。
2. 如果你之前已经以间接经销商的身份加入了合作伙伴中心，请使用全局管理员凭据在[合作伙伴中心仪表板](https://partner.microsoft.com/pcv/dashboard/overview)页面上接受 Microsoft 合作伙伴协议。
可以通过[此处](https://partner.microsoft.com/resources/collection/indirect-reseller-onboarding-for-mpa-in-csp#/)的经销商 CSP 库页面获取特定于间接经销商的信息和资源。
