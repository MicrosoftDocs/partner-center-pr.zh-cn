## <a name="sandbox-support-for-indirect-model-to-enable-end-to-end-api-testing"></a><a name="16"></a>对间接模型的沙盒支持，可实现端到端 API 测试

### <a name="categories"></a>类别

- 日期：2021 年 5 月 25 日
- 功能

### <a name="summary"></a>摘要

宣布正式发布对有权访问沙盒租户的所有云解决方案提供商 (CSP) 间接提供商的间接模型提供沙盒支持。

### <a name="impacted-audience"></a>影响受众

CSP 间接提供商

### <a name="details"></a>详细信息

今天，我们宣布正式发布对间接模型的沙盒支持。 这将使间接提供商能够在部署到生产环境之前，在沙盒中自动测试功能。 以下几点适用： 

- 间接提供商现在可以使用 API 在沙盒中创建和删除测试间接经销商。

- 你可以在沙盒环境中使用间接模型体验进行端到端测试，因为它与生产环境相同。

- 沙盒间接经销商不需要真正的间接经销商来注册。

- 沙盒间接经销商只能在具有测试 MPN 成员资格的沙盒中使用。

### <a name="faq"></a>常见问题解答：

**为什么在创建沙盒间接经销商时没有分配管理员代理角色？**

这是一个已知问题。 我们正在努力解决此问题，并会及时提供更新。

**什么时候能为沙盒间接经销商自动创建管理员代理角色？**

此更新应会在 6 月中旬之前提供。

**如果沙盒间接经销商默认没有创建管理员代理角色，该如何解决这个问题？**

当你创建初始沙盒间接经销商时，记下登录凭据，然后按照以下过程操作。

1. 使用间接经销商沙盒帐户凭据登录到[合作伙伴中心面板](https://partner.microsoft.com/dashboard/)。

2. 选择“设置”，然后选择“帐户设置”。

3. 选择“用户管理”选项卡，然后选择用户的电子邮件。

4. 从下拉菜单中，选择“协助客户的身份”，然后选择“管理员代理”。

5. 选择“更新”。

6. 注销合作伙伴中心，然后重新登录。

7. 选择“CSP”选项卡。在左侧导航栏中，选择“客户”选项卡，会看到已分配此角色。

**作为间接提供商，为什么我不能在自己的国家/地区外创建沙盒间接经销商？**

这是一个已知问题，即将得到解决。

### <a name="next-steps"></a>后续步骤

沙盒间接提供商可以开始创建沙盒间接销售商。 你还可以创建和邀请客户，并与沙盒间接经销商的记录合作伙伴 (POR) 进行交易。 完成测试后，在删除经销商之前，应先删除所有订阅和客户。

### <a name="questions"></a>是否有任何问题?

有关这些产品/服务的问题，请参阅[合作伙伴中心 SDK 和 API](https://www.yammer.com/cloudpartnercommunity/#/threads/inGroup?type=in_group&feedId=6589502)。

## <a name="phase-2-coming-soon-euefta-and-uk-change-of-partner-billing-currency-for-new-commerce-offers"></a><a name="13"></a>第 2 阶段即将推出：欧盟/欧洲自由贸易联盟和英国针对新商业产品/服务的合作伙伴计费货币变更

### <a name="categories"></a>类别

- 日期：2021 年 5 月 17 日
- 功能

### <a name="summary"></a>摘要

即将推出针对新商业产品/服务的欧盟 (EU)/欧洲自由贸易联盟 (EFTA) 和英国 (UK) 合作伙伴计费货币变更第 2 阶段。

### <a name="impacted-audience"></a>影响受众

在欧盟/欧洲自由贸易联盟和英国区域中通过云解决方案提供商 (CSP) 计划进行交易的合作伙伴

### <a name="details"></a>详细信息

在欧盟/欧洲自由贸易联盟和英国区域，参与 CSP 计划的所有新商业产品/服务都将使用合作伙伴计费位置，而不是客户计费位置。 这意味着 Microsoft 将根据合作伙伴位置所用货币，而不是根据其客户位置所用货币对这些合作伙伴进行计费。 我们在 2021 年 1 月 28 日宣布了[第 1 阶段上线](./2021-january.md#15)。 现在，我们已准备好分享有关此转换体验第 2 阶段的更多详细信息。

#### <a name="phase-2"></a>阶段 2

- 第 1 阶段之前已在 CSP 中购买过新的商业产品/服务的现有客户

- 在 2020 年 5 月 11 日当天或之后创建了租户，但第一次购买新的商业产品/服务的现有客户

自 2021 年 7 月中旬开始，如果合作伙伴有客户已于第 1 阶段之前在 CSP 中购买新的商业产品/服务，或者合作伙伴有客户于 2020 年 5 月 11 日当天或之后创建了其租户，Microsoft 会将他们的新商业产品/服务的计费从客户位置所用货币转换到合作伙伴位置所用货币。

第 2 阶段之后，欧盟/欧洲自由贸易联盟和英国区域中的合作伙伴将按合作伙伴位置所用货币为所有客户和所有 CSP 购买计费。

第 2 阶段将分批推出给合作伙伴，这意味着各合作伙伴组织之间第 2 阶段转换的确切开始日期会存在差异。 我们将在未来几周直接与合作伙伴联系，告知与其转换日期相关的详细信息。

请注意，此更改将只影响合作伙伴计费货币，不影响 CSP 中新商业产品/服务的定价。 此更改范围内的新商业产品/服务包括：

- 属于 Azure 计划的 Azure 订阅

- Azure 预留项

- 服务器订阅

- 永久性软件

- CSP 计划中的 Microsoft 商业市场购买

#### <a name="partneraction"></a>合作伙伴操作

对于第 2 阶段，需要合作伙伴执行操作，因为更改将影响现有商业客户发票。

- 使用 Microsoft API 或第三方控制面板供应商 (CPV) 进行任何类型的自动计费的合作伙伴必须进行必要的更改，以适应其客户的货币变化。

- 使用合作伙伴中心或 Azure.com 门户用户界面 (UI) 的合作伙伴必须准备好在第 2 阶段之后更改货币。

#### <a name="partnerbenefits"></a>合作伙伴权益  

对于新的商业体验，此项更改将降低欧盟/欧洲自由贸易联盟和英国区域中用多种货币开票所带来的复杂程度和开销。

- 合作伙伴将收到一张用单一货币显示的合并发票，不再就每个客户位置所用货币单独收到一张发票。  

- 激励付款将与合作伙伴发票采用同一种货币。

- 合作伙伴将注意到因多种货币开票而带来的计费复杂性得到了降低，这将节省帐户对帐所涉及的时间和资源。

- 对于尚未采用新的商业产品/服务的合作伙伴，此更改与之前的合作伙伴计费模型保持一致，使其能够轻松转换到 CSP 中的新商业体验。

可在 Microsoft 合作伙伴网站上的[运营资源库](https://partner.microsoft.com/resources/collection/eu-efta-changes-collection#/)中查看有关此主题的信息（需要登录）。

## <a name="deprecation-of-microsoft-365-business-voice-getting-started-wizard-gsw"></a><a name="12"></a>弃用 Microsoft 365 商务语音入门向导 (GSW)

### <a name="categories"></a>类别

- 日期：2021 年 5 月 14 日
- 功能

### <a name="summary"></a>摘要

Microsoft 365 商务语音入门向导 (GSW) 将于 2021 年 6 月 1 日停用，并将被改进的 Teams 管理中心体验和新的设置文档所取代。

### <a name="impacted-audience"></a>影响受众

销售 Microsoft 365 商务语音通话套餐的美国、英国和加拿大云解决方案提供商 (CSP) 合作伙伴

### <a name="details"></a>详细信息

Microsoft 365 商务语音已帮助改变了中小型企业 (SMB) 的通信方式，同时提高了效率，简化了技术管理流程，并节省了成本。

过去几个月，我们一直在新的语音功能方面投入工作，使客户能够更好地沟通和协作，这些功能包括简化的通话体验、协作通话和支持语音的渠道。

自 2021 年 6 月 1 日起，Microsoft 365 商务语音入门向导 (GSW) 将停用，取而代之的是改进的 Teams 管理中心体验和更深入的全新文档，有助于初始商务语音设置。 这将包括有关电话号码获取、许可证和电话号码分配、呼叫队列和自动助理设置的分步说明。

### <a name="faqs"></a>常见问题解答

#### <a name="who-is-impacted-by-this-change"></a>哪些人会受此更改的影响？

只有新的商务语音客户会受到影响。 自 2021 年 6 月 1 日起，所有新客户都将被定向到新文档。 从 2021 年 6 月 1 日到 6 月 30 日，GSW 仍可供现有客户使用，为客户提供 30 天时间来完成向导，之后即不再可用。

#### <a name="in-what-marketsgeos-was-the-gsw-available"></a>GSW 在哪些市场/GEO 中提供？

在美国、英国和加拿大仅适用于具有商务语音通话套餐的客户

#### <a name="are-there-other-changes-to-business-voice"></a>商务语音是否有其他变更？

否。 这只会影响入门向导功能。 Microsoft 将继续致力于改进 Microsoft 365 商务语音功能，使客户和合作伙伴能够更好地进行沟通和协作。

### <a name="next-steps"></a>后续步骤

如果你是销售商务语音通话套餐的美国、英国或加拿大合作伙伴，我们建议你熟悉新的[分步文档](/microsoftteams/business-voice/set-up-overview)（该文档在 [Microsoft 365 商务语音文档中心](/microsoftteams/business-voice/)提供），准备好为可能需要协助其设置商务语音的客户提供支持。

- [商务语音设置文档](/microsoftteams/business-voice/set-up-overview)
- [Microsoft 365 商务语音文档中心](/microsoftteams/business-voice/)
