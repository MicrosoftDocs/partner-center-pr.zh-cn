# <a name="how-indirect-providers-can-help-their-indirect-resellers-adopt-the-microsoft-partner-agreement-mpa"></a>间接提供商如何帮助其间接经销商采用 (MPA 的 Microsoft 合作伙伴协议) 

**相应的角色**

- 全局管理员
- 管理员代理
- 销售代理

以下问题和解答可帮助间接经销商了解如何以及为什么对 MPA 进行签名。 

云解决方案提供商计划中的间接提供商、直接帐单合作伙伴和间接经销商将会很有用。

### <a name="what-are-the-legal-implications-if-the-indirect-reseller-does-not-sign-the-microsoft-partner-agreement-mpa"></a>如果间接经销商不 (MPA) 签署 Microsoft 合作伙伴协议，会有什么法律含义？ 
 
**答案** 除非 (间接经销商) 对 MPA 进行签名，否则不能在 CSP 计划中使用间接提供程序 (分发服务器) 。 必须为要在其中执行业务的每个 [CSP 区域](regional-authorization-overview.md) 签署 MPA。 对于间接经销商，每个 CSP 区域至少需要一个 MPA 签名，才能遵从该 CSP 区域中任何间接提供程序执行 CSP 交易。 

### <a name="is-mpa-available-in-local-languages"></a>MPA 是否以本地语言提供？
 
**答案** 不。 它仅以英语提供。 

### <a name="i-work-with-multiple-indirect-providers-distributors-in-various-countries-do-i-need-to-sign-the-mpa-in-each-csp-country"></a>我使用多个间接提供程序 (分销商) 在不同国家/地区。 是否需要在每个 CSP 国家/地区对 MPA 进行签名？


**答案** 对于 MPN ID，每个 CSP 区域至少需要一个 MPA 签名。 例如，如果你有一个法国的 MPN ID，并且已在法国和签名的 MPA 中注册了 CSP 间接经销商计划，则你在整个 [CSP 欧洲地区](regional-authorization-overview.md#europe-region-and-market) 都符合要求，无需在其他欧洲国家/地区对 mpa 进行签名。 可以在整个 CSP 欧洲区域中与任何间接提供程序进行处理。 

### <a name="i-am-a-direct-bill-partner-in-france-and-also-transact-with-an-indirect-provider-in-italy-do-i-need-to-sign-two-mpas--one-as-direct-bill-and-one-as-indirect-reseller-"></a>我是法国的直销合作伙伴，也在意大利的间接提供商进行处理，我是否需要将两 MPAs （一种作为直接帐单）和一个间接经销商进行签名？

**答案**  如果已在 CSP 区域签署了直接帐单 MPA，则无需对 CSP 间接经销商 MPA 进行签名。 但是，你需要确保你的 MPN 帐户在合作伙伴中心中，并且与你的间接提供商共享的 MPN ID 位于合作伙伴中心。 

### <a name="do-we-have-to-keep-signing-the-indirect-reseller-mpa-agreement-every-year"></a>我们是否必须每年保持每年的间接经销商 MPA 协议的签名？


**答案** 无需每年进行签名。 若要使用间接提供程序进行处理，必须对每个 CSP 区域至少有一个 MPA，才能保持符合性。


### <a name="if-the-subscriptions-are-not-changing-does-it-matter-if-resellers-sign-the-agreement"></a>如果订阅未更改，请务必要签署协议吗？ 

**答案** 如果间接经销商不对 MPA 进行签名，则会阻止它们通过间接提供程序约束力。 他们的 Microsoft 奖励将受到影响。 

### <a name="will-i-be-off-boarded-from-partner-center-if-i-dont-sign-the-mpa-what-are-the-exact-risks-if-resellers-dont-sign-the-mpa"></a>如果未对 MPA 进行签名，是否会从合作伙伴中心关闭载入？ 如果经销商没有签署 MPA，会有什么风险？

**答案** 间接经销商 (分销商) ，间接经销商将被阻止约束力。 在不久的将来，不会将间接销售商转到任何计划。 

### <a name="my-reseller-says-that-the-mpa-was-already-signed-by-an-employee-who-left-the-company-do-they-need-to-sign-again"></a>我的分销商说，MPA 已由离开公司的员工进行签名。 是否需要再次进行签名？

**答案** 不

### <a name="my-reseller-recently-changed-company-name-but-signed-the-mpa-already-do-they-need-to-sign-again"></a>我的分销商最近更改了公司名称，但已对 MPA 进行签名。 是否需要再次进行签名？

**答案** 不

### <a name="do-we-have-to-communicate-with-our-resellers-about-signing-the-mpa-or-microsoft-will-work-with-resellers-directly-to-sign--mpa"></a>我们是否需要与我们的分销商沟通来签署 MPA，或者 Microsoft 将直接与经销商一起签署 MPA？
 
**答案** 作为间接提供商，你有责任清楚地与所有间接经销商进行沟通，并使他们知道不会对 MPA 进行签名。 还需要为其提供相应的支持和对 MPA 签名的指导。 对于不是由你可以解决的关键问题，你应将其重定向到 Microsoft。

### <a name="my-reseller-believes-they-are-already-enrolled-as-an-indirect-reseller-because-they-have-been-able-to-resell-cloud-subscriptions-through-us-however-they-have-no-option-to-sign-the-mpa-why-is-that"></a>我的分销商认为他们已经注册为间接经销商，因为他们已可以通过我们的身份转售云订阅。 为什么会这样？

现在是时候成为官方 CSP 间接经销商的时候了。
这也适用于已约束力 CSP 的 "顾问" 合作伙伴。 对于间接经销商使用间接提供程序，Microsoft 每个 CSP 区域都需要一个有效的 CSP 间接经销商帐户。

### <a name="how-do-i-know-if-i-have-right-role-in-partner-center-to-sign-mpa"></a>如何实现知道在合作伙伴中心是否有适当的角色来签署 MPA？

**答案** 只有全局管理员才能对 MPA 进行签名。 要了解合作伙伴中心的角色，请执行以下操作：

1. 在合作伙伴中心登录到 CSP 间接经销商帐户，并首先验证是否已注册到 CSP 间接经销商帐户中。 

2. 请切换到 " **用户管理** "，并检查登录帐户是否分配了 "全局管理员" 角色？ 如果不是，则不能用于对 MPA 进行签名。 在 "用户管理" 页上的搜索框中搜索 "全局管理员"，并与他们联系以对 MPA 进行签名。

### <a name="how-do-i-check-if-my-company-has-signed-the-mpa"></a>如何实现检查我的公司是否已对 MPA 进行签名？

**答案** 用 CSP 间接经销商帐户登录到合作伙伴中心。
如果你是 CSP 中的管理代理或销售代理，则可以使用 "协议" 页来验证 Microsoft 合作伙伴协议是否与签名的详细信息一起显示。

:::image type="content" source="images/agreement-new.png" alt-text="在帐户设置中查找协议":::


如果你是全局管理员，请参阅 "CSP 概述" 页。 如果看到绿色横幅，则会设置为与该 CSP 区域中的间接提供程序进行处理。 如果没有，你将看到 " **接受并继续** " 按钮，用于对 MPA 进行签名并激活 CSP 帐户。

:::image type="content" source="images/banner-boxed.png" alt-text="绿色横幅":::

### <a name="i-want-to-sign-mpa-but-i-dont-see-any-place-button-to-accept-terms-to-sign-it"></a>我想要对 MPA 进行签名，但未看到任何位置 (接受术语) 来对其进行签名的按钮？

**答案** 用 CSP 间接经销商帐户凭据登录到合作伙伴中心。 请参阅 CSP 概述页。  如果看不到 " **接受并继续** " 按钮，而且没有绿色横幅告诉你 MPA 已签名，则表示你没有正确的角色对 mpa 进行签名。 你需要联系你的全局管理员对 MPA 进行签名。
