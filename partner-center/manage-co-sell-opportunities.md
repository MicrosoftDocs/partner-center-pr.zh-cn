# <a name="manage-co-sell-opportunities-in-partner-center"></a>管理合作伙伴中心的共同销售机会


**相应的角色**

- MPN 合作伙伴管理员
- 引荐管理员

利用共同销售机会。  共同销售机会是多个参与方的协作，销售满足客户需求的产品。 本文将帮助你了解如何在合作伙伴中心使用共同销售的强大功能来有效地共同销售和获取新客户。

## <a name="types-of-co-sell-opportunities"></a>共同销售机会的类型

**共同销售机会** -这是一个或多个 Microsoft 销售代表积极参与的交易，以帮助解决客户问题。 商机可能源自一个客户帐户，其中，Microsoft 销售代表邀请合作伙伴参与共同销售活动，或从合作伙伴那里需要更多 Microsoft 销售帮助来结束交易。

合作伙伴 **到合作伙伴 (P2P)** –其中一个合作伙伴可以邀请 Microsoft 共同销售生态系统中的其他合作伙伴进行协作。 合作伙伴可以选择邀请 Microsoft 销售代表来帮助完成交易。 只要合作伙伴和 Microsoft 在创建完交易之后就可以得到邀请，只要它在完成之前就会被邀请。  (注意，"完成" 包括赢得、丢失、拒绝和过期) 。

**私人交易** –当合作伙伴决定独立于合作伙伴中心创建的交易时，就会发生这种情况。 合作伙伴可以在创建交易后向 Microsoft 销售团队提供可见性，或邀请 Microsoft 卖方寻找活动的帮助。 此交易的详细信息与 Microsoft 销售团队之间不共享，直到合作伙伴明确选择 "是否希望 Microsoft 卖方查看此交易？" 或通过选择 "确定你需要的 Microsoft 的帮助类型" 一类选项，请求 Microsoft 销售团队帮助。

**合作伙伴导向的交易** -这些交易涉及到合作伙伴决定单独处理某个交易，但却可以向 Microsoft 销售团队成员提供交易细节。 合作伙伴可以在创建交易后邀请 Microsoft 卖方。 当激励合格的解决方案成为交易的一部分时，这些交易将符合交易资格。 如果满足符合要求的解决方案条件，则不需要邀请 Microsoft 卖方进行交易。 您可以通过回答 "是否想要 Microsoft 卖方查看此交易" 这一问题，来做出交易伙伴指导，

**解决方案评估 (SA)** –这些类型的交易只能由解决方案评估业务团队审查的一组合作伙伴创建，以便使用或计划使用 Microsoft 技术对客户执行技术评估。

> [!NOTE]
> 私人交易（即使它们不是真正的协作性），也包括在 "共同销售机会" 选项卡中，以提供一种方便的途径来升级任何渠道，使其能够在任何非终端阶段 **（ (赢得、丢失、拒绝和过期**) ）中处理共同销售机会。

## <a name="navigating-the-co-sell-section"></a>导航共同销售部分

若要访问共同销售部分，请执行以下步骤：

1. 登录到合作伙伴中心[面板](https://partner.microsoft.com/dashboard)。

2. 从左侧导航菜单中选择 " **引用**"，然后选择 " **共同销售" 机会**。

   此部分中将显示三个顶级选项卡：

   - **入站** –此选项卡中提供了从 Microsoft 卖方向合作伙伴发送的用于协作销售的所有机会。

   - **出站** –此选项卡中提供合作伙伴创建的所有商机。合作伙伴可以创建三个不同类别的机会-共同销售、私有和合作伙伴到合作伙伴。

   - **收藏夹** –入站或出站销售机会选项卡中标记为收藏的机会将显示在此选项卡中。在列表视图中单击机会上的心形图标，将任何商机标记为收藏。 只需再次单击相同的图标即可从收藏夹中删除此机会。

## <a name="creating-a-co-sell-opportunity"></a>创建共同销售机会

> [!IMPORTANT]
> 若要与 Microsoft 进行联合销售，合作伙伴需要可供商用的解决方案、投入的销售团队和强大的上市策略。 它们还必须满足特定的附加条件。 有关详细信息，请参阅 [成为共同销售就绪](https://partner.microsoft.com/reach-customers/selling-with-microsoft#become-ready)。

你可以创建三种类型的机会。

### <a name="select-your-customer"></a>选择客户

选择特定交易的客户。 输入其位置，然后键入名称。 建议框将根据搜索条件开始显示建议。 由于许多公司在同一位置有多个业务实体，你可能会看到多个结果。 如果不确定选择哪一种，可以确认哪个 D-N-S 号码属于您的客户。 请确保选择与您正在使用的客户完全匹配的匹配项。

单击 **"选择客户**" 后，系统会提示输入联系人的姓名、电话号码和电子邮件地址，作为此交易的主要联系点。 这些详细信息是可选的，仅当你打算邀请 Microsoft 销售人员帮助你参与服务时，这些详细信息才是必需的。 选择“下一步”。

> [!IMPORTANT]
> 请确保已获得客户同意向 Microsoft 提供其联系信息，以应对交易。 Microsoft 可能会使用此信息直接与客户联系。

### <a name="deal-details"></a>交易详细信息

输入当前交易的详细信息。 下面定义的字段可能会发生变化，因为你正在积极地与客户合作来结束交易。

| **字段名称** | **必需/可选** | **详细信息** |
|-------------|--------|-------|
|**交易名称** | 必需 | 用于在以后标识交易的友好名称。 |
|**位置**| 必需 | 引用的 MPN 位置范围。 如果引用用户是团队的一部分，则具有此位置范围的引用用户可以查看这些引用。 具有全局作用域的引用管理员和引用管理员可以查看引用，而不考虑位置。 创建引用后，不能编辑位置。|
|**估计值** | 必需 | 基于创建交易时可用的信息的交易价值。|
|**估计结束日期**| 必需| 您希望结束客户交易的截止日期。 |
|**CRM ID**| 可选 | 用各自 CRM 中的机会 ID 标记处理跟踪目的。|
|**市场营销活动 ID**| 可选 | 捕获导致交易的营销活动。 如果你标记了来自具有相同 ID 的市场活动的所有交易，则这一归档可帮助你跟踪某一市场活动的投资回报。|
|**说明**| 可选 | 更新所有最新信息，为公司的其他员工提供对同一交易的其他员工的可见性，或者尝试了解该交易的当前状态。 你还可以使用它作为记录的通信，以便与你的公司在 Microsoft 卖方/其他合作伙伴之间进行讨论。|

### <a name="add-team-members"></a>添加团队成员

添加完交易详细信息后，添加将处理此特定交易的员工。 需要输入员工的姓名、电话号码和电子邮件地址。 这些详细信息是必需的，并且你需要至少一个联系人，其中包含为你输入的所有详细信息来创建交易。 即使在创建交易后，也可以更改这些详细信息。 你先前交易中的最近的联系人将显示在右侧，使你能够快速将它们添加到交易中。 对于 P2P 交易，团队可以让公司和公司的员工发送邀请。

### <a name="add-solutions"></a>添加解决方案 () 

在本部分中，你需要提供与将作为此交易一部分的解决方案相关的信息。 这是必需部分，其中必须添加至少一个解决方案才能创建交易。 可以在创建交易后更改解决方案的详细信息。 有多种类型的解决方案可以添加到交易中，如下所述

- **我公司的解决方案：** 这些是公司发布的共同销售就绪解决方案
- **Microsoft：** 这些是 Microsoft 拥有的解决方案
- **其他第三方解决方案：** 这些是由 Microsoft 共同销售生态系统中的其他合作伙伴发布的共同销售就绪解决方案
- **解决方案评估：** 这些是符合条件的合作伙伴可以根据客户需求选择的评估类型

> [!Important]
> 仅可选择一种评估类型用于解决方案评估交易，而不能添加其他解决方案。 选择解决方案评估后，伙伴必须选择要为其创建评估的位置。 这对于正确的激励付款是必需的。

提供解决方案信息后，选择 "下一步" 以转到可在其中确定销售类型的部分。 如果选择解决方案类型的前三个选项中的解决方案，而不是解决方案评估，则可以使用四个选项：

**私人交易**：如果不邀请 Microsoft 并在此步骤中创建订婚，则其类型为私有管道。 Microsoft 卖方无法查看此交易的详细信息。 您可以通过选择 **"确定你需要的 Microsoft 的帮助类型" 中的 "找不到你所需的帮助类型**"**这一时间点上没有所需的帮助** 来创建专用交易。 **不** 是 **"你喜欢 Microsoft 卖方查看此交易吗？"** 的问题。 您始终可以将私人交易升级到合作伙伴 led 或活动的共同销售交易，直到交易到达终端状态 (赢得、丢失) 。

> [!Important]
> 交易注册不适用于私人交易。 使用激励资格解决方案创建专用交易时要格外小心，因为它们不符合合作伙伴中心的交易注册条件。

**合作伙伴导向的交易**：如果想要从 Microsoft 销售团队寻求任何活动帮助，但要使其在你的管道中可见，则可以创建合作伙伴指导的交易。 你可以通过选择 **"确定你需要的 Microsoft 的帮助类型" 中的 "找不到你的帮助类型**" 这一时刻 **不需要帮助** 来创建合作伙伴导向的交易 **是** 的， **"您希望 Microsoft 卖方查看此交易吗？"** 即使 Microsoft 卖方并不积极涉及到帮助合作伙伴，这些交易仍适用于交易注册。 交易注册的要求是一种激励资格解决方案，该解决方案在交易和赢得交易中处于交易状态。

**共同销售交易：** 如果你选择了 **"确定要从 Microsoft 获得的帮助类型**" 之外的任何其他选项，则交易就会成为一项共同销售的交易，其中 Microsoft 卖方可能会帮助你结束交易。 Microsoft 的帮助请求不能保证 Microsoft 卖方会参与交易。 Microsoft 销售代表有14天的时间决定他们是否要参与。 在 "注释" 部分中，请确保确定所需的帮助类型。

合作伙伴 **到合作伙伴 (P2P) 交易**：通过单击邀请合作伙伴链接，你可以邀请其他合作伙伴来处理交易。 下面是创建 P2P 交易的过程。

- **选择合作伙伴：** 单击邀请合作伙伴后，你将能够开始键入合作伙伴名称，以获取与你输入的名称相匹配的合作伙伴的建议列表。 选择你感兴趣的合作伙伴，以填充该合作伙伴的更多详细信息。 你只能搜索 Microsoft 共同销售生态系统中的合作伙伴，并在合作伙伴中心约束力。

- **估计结束日期：** 这是你希望受邀合作伙伴完成其交易部分的日期。 日期是预先填充的，这样您就可以根据需要选择修改日期。 这是必填字段，在创建交易后你邀请的合作伙伴可以对其进行编辑。 创建交易后，不能修改此字段。

- **估计值和货币：** 这是受邀合作伙伴将在整体交易中产生的交易价值。 请确保在此处输入正确的值，以使受邀伙伴能够决定是否要成为交易的一部分。 受邀合作伙伴可以在创建交易后更改此值。 创建交易后，不能修改此字段。

- **说明：** 添加有关邀请合作伙伴成为此交易的一部分的详细信息。 详细信息将帮助受邀合作伙伴决定是否要参与。

- **添加团队：** 添加公司中将使用受邀合作伙伴的员工。 如果受邀伙伴接受交易，则他们可以添加自己的员工，使这两家公司都可以看到整个团队在交易上开展协作。 只有在创建交易之前，才能修改这些详细信息。 在交易数据中输入的员工详细信息将预先填充，以便您可以更轻松地选择使用此特定合作伙伴的员工。

- **添加解决方案：**  添加希望受邀伙伴将其置于表中的解决方案。 至少必须有一个解决方案。 受邀合作伙伴可以在接受邀请后修改解决方案。

- **确定帮助类型：** 确定帮助类型：最后，确定受邀合作伙伴所需的特定帮助。

对想要邀请到此交易的所有合作伙伴重复此操作。 合作伙伴交易合作伙伴还可以将 microsoft 卖方加入到邀请 Microsoft 和合作伙伴来应对交易。 你还可以在创建交易后，邀请 Microsoft 和合作伙伴。

## <a name="responding-to-a-co-sell-opportunity"></a>响应共同销售机会

每个机会将经历自己的生命周期。

### <a name="received-stage"></a>已接收阶段

在此阶段，如果你已收到来自 Microsoft 卖方或 Microsoft 共同销售生态系统中其他合作伙伴的新联销售机会，请查看详细信息，如果你想要详细了解其业务需求，可随时联系客户。 您可以在此阶段执行两项操作。 接受或拒绝引用：

- **接受：** 输入交易的名称，编辑预计的交易价值，并根据您的评论进行估计的购买时间范围。 建立与客户的联系后，应在 " **说明** " 字段中提供信息，以详细了解客户所需的信息。 你可以根据需要在此处输入你的 CRM ID (仅) 的市场营销活动 ID，该 ID 导致了相应的机会，并添加了公司中将要处理此交易的联系人。

- 完成后，选择“下一步”。 我们会将引用转移到 **下一阶段**，这意味着你计划主动与客户合作以满足其需求。 我们还将使用此信息来帮助你在将来查找类似的交易。

- **拒绝**：选择要拒绝交易的原因并添加要包括的任何便笺，然后选择 " **关闭交易**"。 我们会将其存档为 "已 **拒绝** "，并通知 Microsoft 或合作伙伴向你发送此机会。

- 如果在分配的时间内未响应 (目前14天) ，我们会将其存档为 **过期** ，并通知 Microsoft 或向你发送此机会的合作伙伴。

### <a name="accepted-stage"></a>接受阶段

努力与客户达成交易。 如果要更改为已接受的引用提供的任何信息，请选择 " **编辑**"。 然后，你可以更新交易名称、估计购买日期、估计值、注释、CRM ID 和/或营销活动 ID。  你还可以选择 " **添加你的团队** "，为正在处理交易的任何人员提供名称、电话号码和电子邮件地址。 还可以根据客户需求编辑解决方案。

默认情况下，已创建的所有交易都处于接受阶段。

开始处理交易后，您可以通过在交易生命周期中标记销售阶段来提供您的进度的详细信息。 与最初的验收或创建以及最后的赢单或丢单阶段相比，在交易生命周期中有四个阶段，如下所述。 提供这些详细信息是可选的，但强烈建议您共享这些详细信息，以便在共同销售交易中获取有关 Microsoft 销售代表的相应阶段。

:::image type="content" source="images/pscmigration/salesstage.png" alt-text="显示销售阶段可标记的交易生命周期的图像。":::

> [!Note]
> 如果交易是解决方案评估交易，销售阶段会有所不同。 对于解决方案评估交易，标记销售阶段也是 **必需** 的。 只有在所有销售阶段都标记为完成后，"**赢单**" 按钮才会启用。

下表显示了销售阶段，以及由 Microsoft 合作伙伴中心推荐系统确定的非解决方案评估的相对百分比。

|**销售阶段名称**|**销售阶段百分比**|**销售阶段的定义**|
|:----|:-----|:-----|
|创建|10%|创建出站交易。|
|已接受|10%|接受入站交易。|
|Qualified|20%|在继续操作之前，请先确认交易的价值和客户需求。|
|各种|40%|进一步进一步了解如何准备 POC 或正式建议所需的任何其他项目的详细要求。|
|Proposed|60%|根据客户的需求为客户提出正式的提议。|
|协商|80%|根据方案协调最终术语，使其进入最终状态-入选或失去交易。|
|Won|100%|将交易标记为赢单。|

完成后，可以执行两个操作之一，将交易标记为 " **赢单** " 或 " **丢失** " 以报告结果。

公司不需要遵循相同的销售阶段。 这就是合作伙伴中心识别交易销售阶段的方式，如果要使用 API 传递这些值，会自动将公司的各个阶段映射到这些标准阶段。 如果你使用的是合作伙伴中心 UX，则使用表中所示的百分比来标记销售阶段。

> [!Note]
> 对于某些合格的解决方案，选择 "赢单" 后，系统将要求你提供其他信息来注册交易。 Microsoft 将审查你在此处提供的信息，并且在审查过程中可能会要求你提供其他详细信息。 有关详细信息，请参阅[注册交易](register-deals.md)。

> [!Important]
> 如果交易有资格进行交易注册，请确保在交易创建与交易成为赢单之间存在72小时之间的差距。 如果将交易关闭为早于这一赢，可能导致交易注册被拒绝。

只有满足以下所有条件时，才有资格进行交易注册。

- 交易类型为 "共同销售" 或 "伙伴 led"。
- 在交易中有一种激励合格的解决方案。
- 交易的状态为 "赢单"。
- 如果交易的类型为 "共同销售"，则 Microsoft 已接受邀请或将交易标记为 "赢单"。 您可以通过在您的交易细节下面查看 Microsoft 卡来了解 Microsoft 状态。

> [!Important]
> 仅当该客户的合同中清楚地提到了公司名称和交易中符合条件的解决方案时，才注册此交易。

如果交易有资格进行交易注册，则会有另一个里程碑添加到了名为 "交易注册" 的交易生命周期，如下所示。

:::image type="content" source="images/pscmigration/dealregstages.png" alt-text="显示交易生命周期的图像，从此处开始交易注册的位置。":::

通过 "交易生命周期" " **立即注册** " 按钮，可以选择在将交易标记为 "赢单" 或稍后某个时间点后立即注册该交易。
注册交易后，可以从相同的生命周期查看交易验证的进度。 如果你的公司需要执行任何操作，则相应的错误将显示在交易生命周期视图中。 交易验证完成后，交易将进入已关闭状态。

> [!Important]
> 交易评审和最终验证状态仅适用于 IP 共同销售交易。

## <a name="frequently-asked-questions"></a>常见问题

**：.在将交易标记为赢单或丢失后，是否可以对其进行编辑？**

不能，一旦处理进入终端状态，就不能修改。 "过期"、"拒绝"、"成功" 和 "丢失" 是在不能进一步更新交易的情况下的终端状态。 如果要将交易移到这些终端状态，请务必小心。
