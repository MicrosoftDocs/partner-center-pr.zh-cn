# <a name="manage-co-sell-opportunities-in-partner-center"></a>在 合作伙伴中心 中管理联合销售合作伙伴中心


**适当角色**：MPN 合作伙伴管理员|引荐管理员

利用联合销售机会。  联合销售机会是多个参与方协作销售满足客户需求的产品。 本文将帮助你了解如何使用联合销售功能来有效地合作伙伴中心联合销售并获取新客户。

## <a name="types-of-co-sell-opportunities"></a>联合销售机会的类型

**联合销售机会** - 这是一个或多个 Microsoft 销售代表积极参与交易以帮助解决客户问题的地方。 这些机会可能来自客户帐户，其中 Microsoft 销售代表邀请合作伙伴参与联合销售活动，或者来自需要 Microsoft 销售部门更多帮助才能结束交易的合作伙伴。

**合作伙伴到合作伙伴 (P2P)** – 这是一个合作伙伴可以邀请 Microsoft 联合销售生态系统中的其他合作伙伴进行协作的地方。 合作伙伴可以选择邀请 Microsoft 销售代表参与交易，以帮助关闭交易。 即使在创建交易后，也可以邀请合作伙伴和 Microsoft，只要交易完成。  (，完整包括"获胜"、"丢失"、"已拒绝"和"expired.) 

**私人交易** – 当合作伙伴决定独立处理在 合作伙伴中心 中创建的交易时。 合作伙伴可以向 Microsoft 销售团队提供可见性，或邀请 Microsoft 销售人员在创建交易后寻求主动帮助。 在合作伙伴明确选择"是否希望 Microsoft 销售人员查看此交易？"这一问题之前，不会与 Microsoft 销售团队共享交易的详细信息。 或通过选择问题"确定你需要从 Microsoft 获得的帮助类型？"的选项之一来请求 Microsoft 销售团队的帮助。

**合作伙伴导向的交易** -这些交易涉及到合作伙伴决定单独处理某个交易，但却可以向 Microsoft 销售团队成员提供交易细节。 合作伙伴可以在创建交易后邀请 Microsoft 卖方。 当激励合格的解决方案成为交易的一部分时，这些交易将符合交易资格。 如果满足符合要求的解决方案条件，则不需要邀请 Microsoft 卖方进行交易。 您可以通过回答 "是否想要 Microsoft 卖方查看此交易" 这一问题，来做出交易伙伴指导，

**解决方案评估 (SA)** –这些类型的交易只能由解决方案评估业务团队审查的一组合作伙伴创建，以便使用或计划使用 Microsoft 技术对客户执行技术评估。

> [!NOTE]
> 私人交易（即使它们不是真正的协作性），也包括在 "共同销售机会" 选项卡中，以提供一种方便的途径来升级任何渠道，使其能够在任何非终端阶段 **（ (赢得、丢失、拒绝和过期**) ）中处理共同销售机会。

## <a name="navigating-the-co-sell-section"></a>导航共同销售部分

若要访问共同销售部分，请执行以下步骤：

1. 登录到合作伙伴中心[面板](https://partner.microsoft.com/dashboard)。

2. 从左侧导航菜单中选择 " **引用**"，然后选择 " **共同销售" 机会**。

   此部分中将显示三个顶级选项卡：

   - **入站** –此选项卡中提供了从 Microsoft 卖方向合作伙伴发送的用于协作销售的所有机会。

   - **出站** –此选项卡中提供合作伙伴创建的所有商机。合作伙伴可以创建三个不同类别的机会-共同销售、私有和合作伙伴到合作伙伴。

   - **收藏夹** –入站或出站销售机会选项卡中标记为收藏的机会将显示在此选项卡中。在列表视图中单击机会上的心形图标，将任何商机标记为收藏。 只需再次单击相同的图标即可从收藏夹中删除此机会。

## <a name="creating-a-co-sell-opportunity"></a>创建共同销售机会

> [!IMPORTANT]
> 若要与 Microsoft 进行联合销售，合作伙伴需要可供商用的解决方案、投入的销售团队和强大的上市策略。 它们还必须满足特定的附加条件。 有关详细信息，请参阅 [成为共同销售就绪](https://partner.microsoft.com/reach-customers/selling-with-microsoft#become-ready)。

你可以创建三种类型的机会。

### <a name="select-your-customer"></a>选择客户

选择特定交易的客户。 输入其位置，然后键入名称。 建议框将根据搜索条件开始显示建议。 由于许多公司在同一位置有多个业务实体，你可能会看到多个结果。 如果不确定选择哪一种，可以确认哪个 D-N-S 号码属于您的客户。 请确保选择与您正在使用的客户完全匹配的匹配项。

单击 **"选择客户**" 后，系统会提示输入联系人的姓名、电话号码和电子邮件地址，作为此交易的主要联系点。 这些详细信息是可选的，仅当你打算邀请 Microsoft 销售人员帮助你参与服务时，这些详细信息才是必需的。 选择“**下一步**”。

> [!IMPORTANT]
> 请确保已获得客户同意向 Microsoft 提供其联系信息，以应对交易。 Microsoft 可能会使用此信息直接与客户联系。

### <a name="deal-details"></a>交易详细信息

输入当前交易的详细信息。 下面定义的字段可能会发生变化，因为你正在积极地与客户合作来结束交易。

| **字段名称** | **必需/可选** | **详细信息** |
|-------------|--------|-------|
|**交易名称** | 必需 | 用于标识以后的交易的友好名称。 |
|**位置**| 必需 | 引荐的 MPN 位置范围。 具有此位置范围的引荐用户可以查看引荐（如果他们是团队的一部分）。 具有全局范围的引荐管理员和引荐管理员可以查看引荐，而不考虑位置。 创建引荐后，无法编辑位置。|
|**估计值** | 必需 | 交易值基于创建交易时可用的信息。|
|**估计的结束日期**| 必需| 预期与客户达成交易的日期。 |
|**CRM ID**| 可选 | 使用相应 CRM 中商机的 ID 标记交易，以便进行跟踪。|
|**市场营销活动 ID**| 可选 | 捕获导致交易的市场推广活动。 如果你使用同一 ID 标记来自该市场活动的所有交易，则此存档可以帮助你跟踪特定市场活动的 ROI。|
|**备注**| 可选 | 更新所有最新信息，使公司中处理同一交易或尝试了解交易当前状态的其他员工可见。 还可以将此功能用作记录通信，供 Microsoft 销售人员/其他合作伙伴与贵公司进行讨论。|

### <a name="add-team-members"></a>添加团队成员

添加交易详细信息后，添加将处理此特定交易的员工。 需要输入员工的姓名、电话号码和电子邮件地址。 这些详细信息是必需的，你需要至少有一个联系人，并输入所有详细信息来创建交易。 即使在创建交易后，也可以更改这些详细信息。 你先前交易中的最近的联系人将显示在右侧，使你能够快速将它们添加到交易中。 对于 P2P 交易，团队可以让公司和公司的员工发送邀请。

### <a name="add-solutions"></a>添加解决方案 () 

在本部分中，你需要提供与将作为此交易一部分的解决方案相关的信息。 这是必需部分，其中必须添加至少一个解决方案才能创建交易。 可以在创建交易后更改解决方案的详细信息。 有多种类型的解决方案可以添加到交易中，如下所述

- **我公司的解决方案：** 这些是公司发布的共同销售就绪解决方案
- **Microsoft：** 这些是 Microsoft 拥有的解决方案
- **其他第三方解决方案：** 这些是由 Microsoft 共同销售生态系统中的其他合作伙伴发布的共同销售就绪解决方案
- **解决方案评估：** 这些是符合条件的合作伙伴可以根据客户需求选择的评估类型

> [!Important]
> 仅可选择一种评估类型用于解决方案评估交易，而不能添加其他解决方案。 选择解决方案评估后，伙伴必须选择要为其创建评估的位置。 这对于正确的激励付款是必需的。

提供解决方案信息后，选择 "下一步" 以转到可在其中确定销售类型的部分。 如果选择解决方案类型的前三个选项中的解决方案，而不是解决方案评估，则可以使用四个选项：

**私人交易**：如果不邀请 Microsoft 并在此步骤中创建订婚，则其类型为私有管道。 Microsoft 卖方无法查看此交易的详细信息。 您可以通过选择 **"确定你需要的 Microsoft 的帮助类型" 中的 "找不到你所需的帮助类型**"**这一时间点上没有所需的帮助** 来创建专用交易。 对于 **"** 是否希望 **Microsoft 销售人员查看此交易？"，为"否"。** 在交易达到最终状态之前，始终可以将私人交易升级为合作伙伴引导或主动联合销售 (最终状态，) 。

> [!Important]
> 交易注册不适用于私人交易。 使用奖励符合条件的解决方案创建私人交易时请谨慎，因为它们将不符合在 合作伙伴中心。

**合作伙伴导向型** 交易：如果不希望向 Microsoft 销售团队寻求任何主动帮助，但希望让他们了解你的管道，可以创建合作伙伴引导的交易。 对于"确定需要 Microsoft 提供哪种帮助？"的问题，可以在此时选择"不需要帮助"来创建合作伙伴 **引导的交易。** 对于 **"** 是否希望 **Microsoft 销售人员查看此交易？"，选择"是"。** 即使 Microsoft 卖方未主动参与帮助合作伙伴，这些交易也符合交易注册条件。 交易注册的要求是，在交易中具有符合奖励条件的解决方案，并赢得交易状态。

**联合销售交易：** 如果为"确定希望从 Microsoft 获得的帮助类型 **"** 问题选择除默认选项外的任何选项，则交易将变成联合销售交易，其中 Microsoft 卖方可能会帮助你完成交易。 向 Microsoft 请求帮助并不保证 Microsoft 卖方将参与交易。 Microsoft 销售代表有 14 天的时间决定是否要参与。 在"说明"部分中，请务必确定需要的帮助类型。

**合作伙伴到合作伙伴 (P2P) 交易**：可以通过单击"邀请合作伙伴"链接邀请其他合作伙伴参与交易。 下面是创建 P2P 交易的过程。

- **选择合作伙伴：** 单击邀请合作伙伴后，你将能够开始键入合作伙伴名称，以获取与你输入的名称相匹配的合作伙伴的建议列表。 选择你感兴趣的合作伙伴，以填充该合作伙伴的更多详细信息。 你只能搜索 Microsoft 共同销售生态系统中的合作伙伴，并在合作伙伴中心约束力。

- **估计结束日期：** 这是你希望受邀合作伙伴完成其交易部分的日期。 日期是预先填充的，这样您就可以根据需要选择修改日期。 这是必填字段，在创建交易后你邀请的合作伙伴可以对其进行编辑。 创建交易后，不能修改此字段。

- **估计值和货币：** 这是受邀合作伙伴将在整体交易中产生的交易价值。 请确保在此处输入正确的值，以使受邀伙伴能够决定是否要成为交易的一部分。 受邀合作伙伴可以在创建交易后更改此值。 创建交易后，不能修改此字段。

- **说明：** 添加有关邀请合作伙伴成为此交易的一部分的详细信息。 详细信息将帮助受邀合作伙伴决定是否要参与。

- **添加团队：** 添加公司中将使用受邀合作伙伴的员工。 如果受邀伙伴接受交易，则他们可以添加自己的员工，使这两家公司都可以看到整个团队在交易上开展协作。 只有在创建交易之前，才能修改这些详细信息。 在交易数据中输入的员工详细信息将预先填充，以便您可以更轻松地选择使用此特定合作伙伴的员工。

- **添加解决方案：**  添加希望受邀伙伴将其置于表中的解决方案。 至少必须有一个解决方案。 受邀合作伙伴可以在接受邀请后修改解决方案。

- **确定帮助类型：** 确定帮助类型：最后，确定受邀合作伙伴需要的特定帮助。

对要邀请参与此交易的所有合作伙伴重复此操作。 合作伙伴到合作伙伴交易还可以让 Microsoft 销售人员参与，你正在邀请 Microsoft 和合作伙伴参与该交易。 创建交易后，还可以邀请 Microsoft 和合作伙伴。

## <a name="responding-to-a-co-sell-opportunity"></a>响应联合销售机会

每个机会都经历自身的生命周期。

### <a name="received-stage"></a>接收阶段

在此阶段，如果你从 Microsoft 卖方或 Microsoft 联合销售生态系统的其他合作伙伴收到新的联合销售机会，请查看详细信息，并随意与客户联系（如果你想要了解有关其业务需求的详细信息）。 可以在此阶段执行两个操作。 接受或拒绝引荐：

- **接受：** 输入交易名称，根据评审编辑估计的交易值和估计的购买时间范围。 与客户建立联系后，应在"备注"字段中提供相关信息，以进一步说明客户正在查找的内容。 可以选择在此处输入 CRM ID (仅供参考) ，即导致相应机会的市场营销活动 ID，并添加公司中将处理此交易联系人。

- 完成后，选择“下一步”。 我们将引荐移到下一阶段，这意味着你计划与客户积极互动，以解决其需求。 我们还将使用此信息来帮助你在将来找到类似的交易。

- **拒绝**：选择拒绝交易的原因并添加要包含的任何注释，然后选择"**关闭交易"。** 我们会将其存档为 **"已拒绝** "，并通知 Microsoft 或发送了此机会的合作伙伴。

- 如果未在分配的时间内（ (14 天) ，我们会将其存档为"已过期"，并通知 Microsoft 或发送了此机会的合作伙伴。

### <a name="accepted-stage"></a>接受阶段

努力与客户达成交易。 如果要更改为已接受的引用提供的任何信息，请选择 " **编辑**"。 然后，你可以更新交易名称、估计购买日期、估计值、注释、CRM ID 和/或营销活动 ID。  你还可以选择 " **添加你的团队** "，为正在处理交易的任何人员提供名称、电话号码和电子邮件地址。 还可以根据客户需求编辑解决方案。

默认情况下，已创建的所有交易都处于接受阶段。

开始处理交易后，您可以通过在交易生命周期中标记销售阶段来提供您的进度的详细信息。 与最初的验收或创建以及最后的赢单或丢单阶段相比，在交易生命周期中有四个阶段，如下所述。 提供这些详细信息是可选的，但强烈建议您共享这些详细信息，以便在共同销售交易中获取有关 Microsoft 销售代表的相应阶段。

:::image type="content" source="images/pscmigration/salesstage.png" alt-text="显示销售阶段可标记的交易生命周期的图像。":::

> [!Note]
> 如果交易是解决方案评估交易，销售阶段会有所不同。 对于解决方案评估交易，标记销售阶段也是 **必需** 的。 只有在所有销售阶段都标记为完成后，"**赢单**" 按钮才会启用。

下表显示了销售阶段，以及由 Microsoft 合作伙伴中心推荐系统确定的非解决方案评估的相对百分比。

|**销售阶段名称**|**销售阶段百分比**|**销售阶段的定义**|
|:----|:-----|:-----|
|创建|10%|创建出站交易。|
|已接受|10%|接受入站交易。|
|Qualified|20%|在继续操作之前，请先确认交易的价值和客户需求。|
|开发|40%|进一步开发交易，以了解准备 POC 或正式建议所需的任何其他项目的详细要求。|
|Proposed|60%|根据客户的要求向客户提出正式建议。|
|协商|80%|根据建议协商最终条款，以进入最终状态 - 赢得或失去交易。|
|Won|100%|将交易标记为"已赢得"。|

完成后，可以执行这两个操作之一，将交易标记为"成功"或"丢失"以报告结果。

你的公司不需要遵循相同的销售阶段。 这就是合作伙伴中心销售阶段的方法，如果使用 API 传递这些值，则会自动将公司的阶段映射到这些标准阶段。 如果使用 UX 合作伙伴中心，则表中所示的百分比用于标记销售阶段。

> [!Note]
> 对于某些符合条件的解决方案，选择"获胜"后，会要求你提供其他信息以注册交易。 Microsoft 将审查你在此处提供的信息，并且在审查过程中可能会要求你提供其他详细信息。 有关详细信息，请参阅[注册交易](register-deals.md)。

> [!Important]
> 如果交易符合交易注册条件，请确保交易创建和交易标记为"已赢得"之间存在 72 小时差距。 将交易关闭为之前赢得的交易可能会导致交易注册被拒绝。

交易只有在满足以下所有条件时，才有资格进行交易注册。

- 交易类型为联合销售或合作伙伴引导。
- 交易有一个符合奖励条件的解决方案。
- 交易状态为"已赢得"。
- 如果交易类型为联合销售，则 Microsoft 已接受邀请或将交易标记为"已赢得"。 可以通过查看交易详细信息下方的 Microsoft 卡来了解 Microsoft 状态。

> [!Important]
> 只有在与客户的协定中明确提及你的公司名称和交易中符合奖励条件的解决方案时，才注册交易。

如果交易符合交易注册条件，则交易生命周期中将添加另一个里程碑，称为"交易注册"，如下所示。

:::image type="content" source="images/pscmigration/dealregstages.png" alt-text="显示交易生命周期的图像，从此处开始交易注册的位置。":::

通过 "交易生命周期" " **立即注册** " 按钮，可以选择在将交易标记为 "赢单" 或稍后某个时间点后立即注册该交易。
注册交易后，可以从相同的生命周期查看交易验证的进度。 如果你的公司需要执行任何操作，则相应的错误将显示在交易生命周期视图中。 交易验证完成后，交易将进入已关闭状态。

> [!Important]
> 交易评审和最终验证状态仅适用于 IP 共同销售交易。

### <a name="closed-stage"></a>已关闭阶段

这是所有机会的最后阶段。 可以查看在关闭阶段中处于"获胜"、"丢失"、"已拒绝"和 **"已过期**"状态的所有交易。  在此阶段不能执行任何操作。

## <a name="frequently-asked-questions"></a>常见问题

**问。交易标记为"已赢得"或"已丢失"后，可以对其进行编辑吗？**

否，交易进入终端状态后无法修改。 "过期"、"拒绝"、"获胜"和"丢失"是终端状态，无法进一步更新交易。 将交易移动到这些终端状态之一时，请谨慎操作。

**问。谁从公司收到电子邮件通知合作伙伴中心？**

以下工作流说明了如何将电子邮件从合作伙伴中心引荐系统发送给合作伙伴，以用于新合作伙伴入站引荐。

:::image type="content" source="images/pscmigration/emaillogic.png" alt-text="显示如何向合作伙伴发送新入站引荐的电子邮件的逻辑的图像。":::

**问。哪种类型的奖励符合交易注册条件？**

只有 IP 奖励才有资格进行交易注册。 其中包括：

- Biz 应用高级版
- Biz 应用标准版
