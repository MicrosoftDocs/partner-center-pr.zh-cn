# <a name="explore-the-azure-and-cloud-products-benefits-area-in-partner-center"></a>了解 Azure 和云产品权益区域合作伙伴中心

**适当的角色**：全局管理员|MPN 合作伙伴管理员

本文介绍 Azure 和云中合作伙伴可用的特定 Azure 和云合作伙伴中心。 还将了解如何激活某些类型的 Azure 和云订阅。
 
## <a name="overview"></a>概述

作为 Microsoft 合作伙伴网络的合作伙伴，你可能有权访问不同领域的许多技术和市场销售权益。 如果你已分配到 合作伙伴中心 中的上述角色之一，可以直接从仪表板中浏览合作伙伴中心优势。

> [!NOTE]
> 若要访问合作伙伴中心"权益"区域，只需登录到 合作伙伴中心 [仪表板](https://partner.microsoft.com/dashboard)，然后从左侧导航菜单中选择 **MPN。** 然后， **从菜单中** 选择"权益"，然后选择要浏览的特定权益选项或区域。

可以使用的一个权益选项是 Azure & **云产品**。 从"权益&菜单中选择 **"Azure** 合作伙伴中心 **云** "时，将显示 **"Azure &云产品"** 页。

此页显示以下详细信息：

- 各种 Azure &云产品订阅权益，无论是基于许可证的订阅还是基于使用情况的订阅
- 为每个产品分配的许可证数量
- 预配了特定权益产品的产品/服务类型
- 权益类型 (核心类型还是增量) 
- 特定权益的过期日期

:::image type="content" source="images/benefits/mpn-benefits-azure-cloud-1.png" alt-text="列出可用的订阅权益的 Azure 和云产品页。" border="false":::

## <a name="usage-based-vs-license-based-subscriptions"></a>基于使用情况的订阅与基于许可证的订阅

**Azure & 云产品** 页列出了为你预配的两种类型的订阅权益：
- 基于使用情况的订阅 (Microsoft Azure 信用)  
- 基于许可证的订阅 (如 Office 365) 

### <a name="usage-based-subscriptions-with-two-types-of-microsoft-azure-credit"></a>使用两种类型的 Microsoft Azure 信用额度的基于使用情况的订阅

可以接收两种类型的基于使用情况的订阅，具体取决于 Microsoft 合作伙伴网络成员身份可用的活动服务。 此类活动的产品/服务可能包括 Microsoft Action Pack、银资格或金牌资格。 

可以为这些产品/服务设置的两种基于使用量的订阅包括 *azure 每月信用额度* 和/或 *azure 批量 (每年) 信用额度*：

- **预配 Azure 每月信用额度：** 这可能显示为类似于 $ 100/month (或本地货币的等效项) <br/>
*and/or*
- **针对 Azure bulk (每年) 信用预配：** 这可能显示为 $ 6000/year、$ 12000/year、/或 $ 13000/year (或它们的每个等价) 本地货币

若要在合作伙伴中心激活 Microsoft Azure 订阅，需要为公司 (或来宾用户) 中的用户分配访问权限。 用户必须已在 Azure Active Directory 租户中创建了一个用户条目。 然后，按如下步骤操作：

1. 确保用户在 Azure AD 租户中拥有用户条目后，登录到合作伙伴中心 [仪表板](https://partner.microsoft.com/dashboard)。

1. 从左侧导航菜单中选择 " **MPN** "，然后依次选择 " **权益**"、" **Azure" 和 "云**"。

1. 选择右侧的向下箭头，查看要激活的基于使用情况的订阅的 **详细信息** 列。 

1. ) 下面 (在 "合作伙伴中心 **Azure 和云产品**" 页上找到 "**用户分配**" 区域。 然后，选择 " **分配用户** " 选项，为该用户分配所需的基于使用情况的订阅权益。 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-2.png" alt-text="显示用户分配区域的 Azure 和云产品页。" border="false":::

- 如果每月分配 100 美元的 Azure 每月信用额度 (或以本地货币) 表示的等效额度，请注意，你随时都可以在此订阅中添加或删除用户。

- 将 Azure 批量分配 (额度) ，请注意，"删除"选项在 合作伙伴中心。  相反，需要在注册年份将批量额度转移给其他人。 Azure 批量 () 额度订阅也无法与其他许可、预付或信用额度套餐 (例如 Microsoft 企业协议、云解决方案提供商、Azure 开放许可、Azure Pass、Microsoft Action Pack 等) 结合使用。

### <a name="license-based-subscriptions"></a>基于许可证的订阅

在"Azure 和云产品"页的"基于许可证的订阅"部分中，可以找到需要激活产品密钥的所有云产品，例如 Microsoft O365、Microsoft Dynamics、Microsoft Intune、Azure AD basic、Power BI 等。这些产品基于作为合作伙伴在产品/服务中提供的活动产品/服务Microsoft 合作伙伴网络。 此类产品/服务可能包括 Microsoft Action Pack、银级资格成员身份或黄金资格成员身份。

激活基于许可证的订阅： 

1. 在"Azure 和云产品"页上找到特定订阅。 

1. 选择右侧下拉箭头" **查看该订阅** 的详细信息"列。 

1. 阅读后续页上的说明。 这样做会告知你有关如何兑换密钥的更多信息。 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-3.png" alt-text="包含视图详细信息的 Azure 和云产品页。" border="false":::


1. 准备好兑换密钥后，请选择本页底部的 **"** 激活演练"选项。
   - 如果要激活新订阅或现有订阅已禁用/已过期，请打开专用或 **incognito** 浏览器，然后复制并粘贴以下链接，并按照说明操作：

      [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
      
   - 如果希望延长结束日期或将更多座位/许可证添加到现有订阅副本，并将以下链接粘贴到 **专用或 incognito 浏览器窗口** 中，并按照说明进行操作：
 
      [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

> [!Note]
> 兑换密钥后，不能再次兑换同一密钥。 另外，请注意以下要点：
>
> - 确认是否需要添加更多许可证 (座位) 、数量和是否延长结束日期。 如果选择添加更多许可证 (座位) 并且 **不** 会延长结束日期，则系统将提供少于12个月的活动订阅。 
>
> - 请确保已登录到正确的凭据，并且要在正确的 Azure Active Directory (AAD) 租户上激活密钥。 始终使用私有/incognito 浏览器窗口登录，以确保在正确租户上激活密钥。
>
> - Microsoft 合作伙伴中心中显示的权益到期日期表示对该权益执行使用权限的最后日期。 此日期可能不同于 M365 管理中心门户中列出的订阅结束日期。 这可能是由多个因素引起的，例如，福利激活日期、与密钥关联的许可证的数量，以及在管理门户中添加或扩展的许可证。

## <a name="extend-the-end-date-or-add-more-licenses-to-an-existing-subscription"></a>延长结束日期或向现有订阅添加更多许可证

按照以下步骤为现有订阅添加更多许可证或延长结束日期：

1. 将以下链接复制并粘贴到 **专用或 incognito 浏览器窗口** 中。

   [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

1. 登录到 **M365 管理中心门户** 时，请确保以正确的 Azure Active Directory 租户登录。 然后从列表中选择特定的云订阅，然后执行以下 (之一来续订订阅或添加更多的许可证) ：

   - 若要续订现有订阅，请在 "许可证" 区域中选择 " **延长结束日期** " 选项。 延长结束日期将增加订阅的持续时间，同时保持许可证/座位数量不变。 

   - 若要向现有订阅添加更多许可证 (座位) ，请选择 "许可证" 区域中的 " **添加许可证** " 选项。 如果还想要续订现有的订阅，请确保未选择 " **添加许可证** "。 选择 " **添加许可证** " 将减少订阅持续时间并增加订阅中的许可证数量。

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-4.png" alt-text="显示如何添加许可证的 Azure 和云产品页面。" border="false":::

1. 在侧面板中，选择 " **兑换产品密钥**"。 

1. 输入产品密钥，然后选择 " **验证** " 以验证密钥详细信息。 

1. 选择 " **兑换** " 完成该过程。

   > [!NOTE]
   > 每个产品密钥向用户提供12个月的服务，即你兑换产品密钥的日期。 但是，如果你的现有用户的续订日期较早，则会为你提供两个日期，使你可以更轻松地根据你的所有用户的单个结束日期管理你的订阅。 计算期间，不会添加或丢失时间。 用于计算两个日期的计算结果为：
   >
   > **计算：** \[ (初始用户) x B (订阅上剩余的时间) \+ C (新用户) x (12 个月) \] ÷总用户数 = 新的时间。

## <a name="activate-a-subscription-for-the-first-time"></a>第一次激活订阅

如果你是第一次激活订阅，请执行以下步骤。 如果你的订阅处于禁用或过期状态，你还可以使用这些步骤：

1. 将以下链接复制并粘贴到 **专用或 incognito 浏览器窗口** 中。
   
   [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
1. 请确保已登录到正确的 Azure Active Directory 租户帐户。 登录后，请验证租户详细信息。

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-5.png" alt-text="显示在何处添加密钥的 Azure 和云产品页面。" border="false":::

1. 输入从合作伙伴中心获取的25位数的密钥，然后选择 " **下一步**"。

2. 验证显示的产品详细信息、许可证数量和订阅持续时间。 准备就绪后，选择" **兑换** "以激活订阅。

   新激活的订阅将在 M365 管理中心的"产品"选项卡中提供。 

## <a name="licensing-scenarios"></a>许可方案

本部分介绍与 Azure 和云产品关联的订阅的一些示例许可方案。 在这些情况下，你将了解如何计算订阅结束日期。

### <a name="scenario-1"></a>方案 1

如果订阅上有 100 个有效许可证，并且尝试使用具有 25 个许可证的密钥来延长结束日期，则对于具有 100 个许可证的订阅，结束日期将延长 3 (3) 个月。

### <a name="scenario-2"></a>方案 2

如果在订阅上激活了 100 个许可证，并且尝试使用具有 100 个许可证的密钥来延长结束日期，则结束日期将延长 12 个月。

### <a name="scenario-3"></a>方案 3

如果订阅上有五个 (5) 许可证处于活动状态，并且尝试使用具有 10 个许可证的密钥来延长结束日期，则对于具有五个 (5) 许可证的订阅，结束日期将延长 24 个月 (或席位) 。

### <a name="scenario-4"></a>方案 4

如果尝试使用活动许可证/席位数超过 3 倍的密钥来延长结束日期，将显示以下错误消息。 此消息是设计使的，因为订阅结束日期最多可以延长 3 年。
